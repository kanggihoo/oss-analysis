`pi` 프로젝트는 AI 기반 coding agents를 위한 모듈식 생태계를 제공하도록 설계된 npm workspaces 모노레포로 구성되어 있습니다. AI provider 추상화, 에이전트 로직, 터미널 렌더링, 기본 CLI 구현을 각각 별도의 패키지로 분리합니다.

## 패키지 아키텍처

모노레포는 루트 `package-lock.json`의 `workspaces` 필드를 통해 관리됩니다 [package-lock.json:10-17](). 네 개의 핵심 패키지와 예제 및 확장을 위한 여러 보조 workspaces를 포함합니다 [package-lock.json:12-16]().

### 핵심 패키지

| 패키지 | 이름 | 설명 |
| :--- | :--- | :--- |
| `packages/ai` | `@earendil-works/pi-ai` | 자동 모델 검색과 provider 구성을 갖춘 통합 LLM API입니다 [packages/ai/package.json:2-4](). |
| `packages/agent` | `@earendil-works/pi-agent-core` | transport 추상화, 상태 관리, attachment 지원을 갖춘 범용 에이전트입니다 [packages/agent/package.json:2-4](). |
| `packages/tui` | `@earendil-works/pi-tui` | 효율적인 텍스트 기반 애플리케이션을 위한 차등 렌더링 기반 Terminal User Interface 라이브러리입니다 [packages/tui/package.json:2-4](). |
| `packages/coding-agent` | `@earendil-works/pi-coding-agent` | read, bash, edit, write 도구와 세션 관리를 갖춘 coding agent CLI입니다 [packages/coding-agent/package.json:2-4](). |

### 의존성 그래프

## 빌드 파이프라인

빌드 시스템은 TypeScript 컴파일에 `tsgo`를 사용하고, binary 배포에는 `bun`을 사용합니다.

### TypeScript 컴파일 (`tsgo`)
대부분의 패키지는 배포 파일을 빌드하기 위해 `tsgo`를 사용합니다. 
- **Clean**: `shx rm -rf dist` [packages/coding-agent/package.json:29]().
- **Build**: 패키지는 `tsgo -p tsconfig.build.json`을 실행합니다 [packages/ai/package.json:65](), [packages/tui/package.json:9](), [packages/agent/package.json:25]().
- **Model Generation**: `ai` 패키지는 생성된 모델 메타데이터를 만들기 위해 pre-build 단계인 `generate-models`와 `generate-image-models`를 포함합니다 [packages/ai/package.json:63-65]().

### Binary 배포 (`bun compile`)
Standalone `pi` binary는 Bun을 사용해 `coding-agent` 패키지를 컴파일하여 생성됩니다.
- **Process**: `build:binary` 스크립트는 모든 의존성(`tui`, `ai`, `agent`)을 먼저 빌드한 다음 CLI와 workers를 컴파일합니다 [packages/coding-agent/package.json:31]().
- **Command**: `bun build --compile ./dist/bun/cli.js ./src/utils/image-resize-worker.ts --outfile dist/pi` [packages/coding-agent/package.json:31]().

### Asset 관리
컴파일 후 themes, PNGs, HTML templates 같은 assets가 `dist` 디렉터리로 복사됩니다. 여기에는 이미지 처리에 필요한 `photon_rs_bg.wasm` 파일도 포함됩니다 [packages/coding-agent/package.json:32-33]().

출처: [packages/coding-agent/package.json:29-37](), [packages/ai/package.json:61-68](), [packages/tui/package.json:7-12]()

## 배포와 CI/CD

이 프로젝트는 release 관리를 위해 특수 스크립트와 GitHub Actions를 사용합니다.

### 버전 관리 정책
모노레포는 lockstep versioning 정책을 따릅니다. `scripts/sync-versions.js`를 사용해 모든 패키지의 버전을 동시에 업데이트합니다 [scripts/release.mjs:95]().

### Release Workflow
`scripts/release.mjs` 유틸리티는 release cycle을 자동화합니다.
1. 모든 패키지의 버전을 올립니다 [scripts/release.mjs:80-97]().
2. `CHANGELOG.md` 파일을 업데이트합니다 [scripts/release.mjs:107-126]().
3. release artifacts(models와 shrinkwrap)를 재생성합니다 [scripts/release.mjs:168-172]().
4. release를 commit하고 tag를 생성합니다 [scripts/release.mjs:180-184]().

## 스크립트 유틸리티

`scripts/` 디렉터리에는 필수 유지보수 유틸리티가 포함되어 있습니다.
- **`local-release.mjs`**: 저장소 밖에서 로컬 release 테스트를 할 수 있도록 publish 가능한 패키지를 빌드하고 tarball로 패킹합니다 [scripts/local-release.mjs:15-28]().
- **`generate-coding-agent-shrinkwrap.mjs`**: root lockfile에서 의존성을 해석하여 `coding-agent` 패키지용 특수 `npm-shrinkwrap.json`을 생성합니다 [scripts/generate-coding-agent-shrinkwrap.mjs:7-12]().
- **`check-pinned-deps.mjs`**: 모든 외부 의존성이 정확한(pinned) 버전을 사용하는지 보장합니다 [scripts/check-pinned-deps.mjs:4-5]().
- **`check-ts-relative-imports.mjs`**: build integrity를 유지하기 위해 `.ts` 파일에서 상대 `.js` imports 사용을 방지합니다 [scripts/check-ts-relative-imports.mjs:70-74]().
- **`publish.mjs`**: lockstep versioning이 유지되는지 보장하면서 패키지를 검증하고 npm에 publish합니다 [scripts/publish.mjs:85-90]().
