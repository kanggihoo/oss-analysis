`pi-tui` 패키지는 여러 줄 `Editor`와 한 줄 `Input` 컴포넌트로 구성된 견고한 텍스트 입력 시스템을 제공합니다. 이 컴포넌트들은 그래핌 인식 커서 이동, undo/redo 기록, Emacs 스타일 kill ring 작업을 위한 공통 코어를 공유합니다.

## 입력 컴포넌트

TUI 아키텍처는 한 줄 입력과 여러 줄 입력의 관심사를 두 개의 주요 클래스로 분리하며, 두 클래스 모두 `Focusable` 인터페이스를 구현합니다 [packages/tui/src/tui.ts:96-97]().

### 한 줄 Input
`Input` 클래스는 표준 한 줄 텍스트 필드를 제공합니다 [packages/tui/src/components/input.ts:19](). 텍스트가 컴포넌트 너비를 초과할 때 `sliceByColumn`을 통한 가로 스크롤을 지원합니다 [packages/tui/src/components/input.ts:6](). 또한 bracketed paste mode를 지원하며, buffering input between `\x1b[200~` and `\x1b[201~` markers [packages/tui/src/components/input.ts:48-83]() 방식으로 입력을 버퍼링합니다.

### 여러 줄 Editor
`Editor` 클래스는 word-wrapping, 기록 탐색, 통합 autocomplete를 지원하는 복잡한 여러 줄 텍스트 입력을 처리합니다 [packages/tui/src/components/editor.ts:188-210](). `lines`, `cursorLine`, `cursorCol`을 포함한 상태를 유지합니다 [packages/tui/src/components/editor.ts:189-191](). coding agent에서는 `CustomEditor`가 이를 확장하여 `AppKeybinding` 핸들러를 통해 인터럽트하거나 종료하는 등 애플리케이션 수준의 동작을 처리합니다 [packages/coding-agent/src/modes/interactive/components/custom-editor.ts:7-16]().

### 데이터 흐름과 입력 처리
두 컴포넌트 모두 `handleInput(data: string)`을 통해 원시 터미널 데이터를 처리합니다.

| 기능 | Input 클래스 | Editor 클래스 |
| :--- | :--- | :--- |
| **Undo Stack** | `UndoStack<InputState>` [packages/tui/src/components/input.ts:37]() | `UndoStack<EditorState>` [packages/tui/src/components/editor.ts:210]() |
| **Kill Ring** | `KillRing` [packages/tui/src/components/input.ts:33]() | `KillRing` [packages/tui/src/components/editor.ts:209]() |
| **Paste Mode** | Bracketed Paste [packages/tui/src/components/input.ts:53]() | 마커 인식 세그먼트화 [packages/tui/src/components/editor.ts:32]() |
| **탐색** | 가로만 | 세로 및 가로 |

## 그래핌 및 마커 인식

정확한 커서 이동과 word-wrapping을 보장하기 위해 TUI는 그래핌 클러스터 인식에 `Intl.Segmenter`를 사용합니다 [packages/tui/src/utils.ts:4-10]().

### Paste 마커
`Editor`는 특수한 `segmentWithMarkers` 함수를 지원합니다. 이 함수는 `PASTE_MARKER_REGEX`를 사용하여 "paste markers"(예: `[paste #1 +123 lines]`)를 단일 원자적 세그먼트로 병합합니다 [packages/tui/src/components/editor.ts:15-48](). 이를 통해 커서가 마커 중간으로 들어가지 않도록 하고, 삭제나 wrapping 중에 전체 마커가 단일 단위로 취급되도록 보장합니다 [packages/tui/src/components/editor.ts:21-23]().

### Word-Wrap 구현
`wordWrapLine` 함수는 시각적 렌더링을 위해 하나의 논리적 줄을 여러 `TextChunk` 객체로 분할합니다 [packages/tui/src/components/editor.ts:107](). 공백에서 wrapping을 시도하지만, 단일 단어가 터미널 너비를 초과하면 문자 단위 wrapping으로 대체합니다 [packages/tui/src/components/editor.ts:136-152]().

## 키 바인딩과 의미론적 동작

`KeybindingsManager`는 물리적 키 시퀀스를 의미론적 동작 ID(예: `tui.editor.cursorWordLeft`)에 매핑합니다 [packages/tui/src/keybindings.ts:155]().

### 네임스페이스가 적용된 동작
세분화된 override를 허용하기 위해 동작은 네임스페이스별로 구성됩니다.
- `tui.editor.*`: 탐색, 삭제, 기록 [packages/tui/src/keybindings.ts:9-29]().
- `tui.input.*`: 제출 및 tab-completion [packages/tui/src/keybindings.ts:31-34]().
- `tui.select.*`: 선택 목록 탐색 [packages/tui/src/keybindings.ts:36-41]().

### Kitty Keyboard Protocol
시스템은 **Kitty Keyboard Protocol**을 지원하여 `Ctrl+Enter`와 `Enter` 같은 키를 구분하고 비라틴 레이아웃의 "base" 키를 보고할 수 있게 합니다 [packages/tui/src/keys.ts:4-10](). `matchesKey` 함수는 shifted 키와 대체 레이아웃을 포함한 복잡한 CSI-u 시퀀스를 처리합니다 [packages/tui/src/keys.ts:14-15](). 예를 들어 기본 레이아웃 키가 'c'이면 키릴 문자를 누를 때도 `ctrl+c`와 매칭할 수 있습니다 [packages/tui/test/keys.test.ts:47-53]().

## Autocomplete 통합

`Editor`는 `AutocompleteProvider`와 통합되어 overlay `SelectList`에 제안을 표시합니다 [packages/tui/src/components/editor.ts:1-9]().

### CombinedAutocompleteProvider
기본 구현인 `CombinedAutocompleteProvider` [packages/tui/src/autocomplete.ts:270]()는 다음을 처리합니다.
1. **Slash Commands**: `SlashCommand` 인터페이스를 통해 등록됩니다 [packages/tui/src/autocomplete.ts:227]().
2. **파일 경로**: 고성능 `.gitignore` 인식 파일 탐색을 위해 `walkDirectoryWithFd`를 통해 `fd` 유틸리티를 사용합니다 [packages/tui/src/autocomplete.ts:124-150](). 상대 경로와 절대 경로를 모두 처리하며 `@` 또는 `"` 접두사를 지원합니다 [packages/tui/src/autocomplete.ts:94-105]().

### Trigger 메커니즘
- **명시적**: `tui.input.tab` 키 바인딩에 의해 trigger됩니다 [packages/tui/src/keybindings.ts:120]().
- **암시적**: `@`(파일) 또는 `/`(명령)를 입력하면 trigger됩니다 [packages/tui/src/autocomplete.ts:94-105]().

**출처:** [packages/tui/src/autocomplete.ts:241-270](), [packages/tui/src/components/select-list.ts:40-58](), [packages/tui/test/autocomplete.test.ts:57-115]()

## Undo Stack과 Kill Ring

### Undo/Redo
`UndoStack`은 컴포넌트 상태의 기록을 유지합니다. `Editor`와 `Input` 모두 **undo coalescing**을 사용합니다. 즉, 과도한 undo 단계를 피하기 위해 연속된 문자 삽입을 단일 `type-word` 동작으로 취급합니다 [packages/tui/src/components/input.ts:213-218]().

### Kill Ring (Emacs 스타일)
`KillRing`은 고급 텍스트 조작을 가능하게 합니다.
- **Kill**: 텍스트 삭제(예: `tui.editor.deleteToLineEnd`를 통한 `Ctrl+K`)는 삭제된 텍스트를 ring에 push합니다 [packages/tui/src/components/input.ts:132-135]().
- **Yank**: ring에서 가장 최근 항목을 붙여넣습니다 [packages/tui/src/components/input.ts:138-141]().
- **Yank-Pop**: yank 직후 ring의 이전 항목들을 순환합니다 [packages/tui/src/components/input.ts:142-145]().

**출처:** [packages/tui/src/components/input.ts:33-37](), [packages/tui/src/components/input.ts:132-145](), [packages/tui/src/components/editor.ts:209-210]()
