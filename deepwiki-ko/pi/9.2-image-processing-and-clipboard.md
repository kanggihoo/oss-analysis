`pi` 코드베이스는 크기 조정, 형식 변환, EXIF orientation 보정, 플랫폼 간 클립보드 이미지 캡처에 초점을 맞춘 포괄적인 이미지 처리 유틸리티를 제공한다. 이 유틸리티들은 이미지가 터미널 표시용으로 올바르게 처리되고, provider payload 제한을 준수하며, 원활한 크로스 플랫폼 붙여넣기 호환성을 가능하게 한다.

## 이미지 처리 파이프라인

이미지 처리는 핵심 작업에 **Photon**(WebAssembly로 컴파일된 Rust 기반 이미지 처리 라이브러리)을 활용한다. 이 시스템은 시각적 충실도를 유지하면서 이미지가 LLM provider의 token 및 byte 제한 안에 들어가도록 견고한 크기 조정 전략을 구현한다.

### 크기 조정과 최적화

주요 진입점인 `resizeImage`는 지능적인 다단계 전략을 사용해 이미지를 비동기적으로 크기 조정한다. TUI 이벤트 루프를 blocking하지 않도록 worker thread에서 Photon을 실행한다 [packages/coding-agent/src/utils/image-resize.ts:78-84]().

**주요 구현 세부 사항:**

1.  **Worker Thread 실행**:
    - `resizeImage` 함수는 `image-resize-worker.ts`를 사용해 `Worker` 생성을 시도한다 [packages/coding-agent/src/utils/image-resize.ts:91-103]().
    - 데이터를 worker로 효율적으로 이동하기 위해 `toTransferableBytes`를 사용한다 [packages/coding-agent/src/utils/image-resize.ts:11-15]().
    - worker 초기화가 실패하면(일부 Bun 컴파일 바이너리 레이아웃에서 흔함) `resizeImageInProcess`로 fallback한다 [packages/coding-agent/src/utils/image-resize.ts:105-109]().

2.  **제약 조건 적용**:
    - processor는 이미지가 `maxWidth`와 `maxHeight` 안에 들어가도록 scale한다 [packages/coding-agent/src/utils/image-resize.ts:85-89]().
    - 결과 인코딩 이미지가 `maxBytes`를 초과하면, 시스템은 예산을 만족할 때까지 품질(JPEG의 경우) 또는 치수를 반복적으로 줄인다.

3.  **치수 매핑**:
    - `formatDimensionNote`는 LLM에 scale factor를 설명하는 문자열을 생성한다: `[Image: original ...x..., displayed at ...x... Multiply coordinates by ... to map to original image.]` [packages/coding-agent/src/utils/image-resize.ts:116-123]().

### EXIF Orientation 보정

크기 조정 또는 표시 전에 이미지는 `applyExifOrientation`으로 처리된다. 이 유틸리티는 TIFF/JPEG/WebP 헤더를 읽어 orientation tag를 찾고 Photon을 사용해 필요한 flip 또는 rotation을 적용한다 [packages/coding-agent/src/utils/exif-orientation.ts:147-183]().

- **JPEG**: `0xffe1` marker와 `Exif\0\0` header를 검색한다 [packages/coding-agent/src/utils/exif-orientation.ts:39-63]().
- **WebP**: RIFF container 안에서 `EXIF` chunk를 검색한다 [packages/coding-agent/src/utils/exif-orientation.ts:65-85]().

## 클립보드 이미지 캡처와 관리

클립보드 시스템은 로컬 및 원격(SSH) 세션 전반에서 동작하도록 설계되었으며, Linux(X11/Wayland), macOS, Windows(WSL 포함)를 지원한다.

### 구현 전략

핵심 함수 `readClipboardImage`는 감지된 환경에 따라 전략을 선택한다.

-   **Wayland**: 선호 MIME type을 찾기 위해 `wl-paste --list-types`를 사용하고, 데이터를 추출하기 위해 `wl-paste --type ...`를 사용한다 [packages/coding-agent/src/utils/clipboard-image.ts:118-141]().
-   **WSL (Windows Subsystem for Linux)**: Linux 도구가 Windows 클립보드에 직접 접근할 수 없으므로, PowerShell script를 사용해 `[System.Windows.Forms.Clipboard]::GetImage()`를 호출하고 임시 파일에 저장한다 [packages/coding-agent/src/utils/clipboard-image.ts:161-201]().
-   **X11**: `xclip -selection clipboard -t TARGETS`를 사용한다 [packages/coding-agent/src/utils/clipboard-image.ts:213-238]().
-   **Native Addon**: macOS와 Windows에서는 `@mariozechner/clipboard`(Rust 기반 native binding)를 사용한다 [packages/coding-agent/src/utils/clipboard-native.ts:17-32]().

### BMP에서 PNG로 변환

Windows/WSL에서는 이미지가 클립보드에 BMP로 저장되는 경우가 많다. 시스템은 크로스 플랫폼 호환성과 터미널 지원을 보장하기 위해 Photon의 `get_bytes()` 메서드를 사용해 이를 PNG로 자동 변환한다 [packages/coding-agent/src/utils/clipboard-image.ts:71-87]().

### 텍스트 복사와 OSC 52

텍스트의 경우 `copyToClipboard`는 먼저 native 도구(`pbcopy`, `clip.exe`, `wl-copy`)를 시도한다 [packages/coding-agent/src/utils/clipboard.ts:66-113](). 원격 세션(SSH/Mosh)에 있거나 native 도구가 실패하면, 터미널에 **OSC 52** escape sequence를 내보내 로컬 터미널 emulator가 클립보드 쓰기를 처리하게 한다 [packages/coding-agent/src/utils/clipboard.ts:119-122]().

## 형식 변환과 터미널 호환성

### Kitty Graphics Protocol 지원
Kitty terminal protocol은 PNG 형식을 요구한다. `convertToPng` 유틸리티(`image-convert.ts` 안)는 PNG가 아닌 모든 base64 이미지 데이터가 터미널로 전송되기 전에 Photon을 사용해 변환되도록 보장한다 [packages/coding-agent/src/utils/image-convert.ts:8-41]().

### CLI 파일 처리
사용자가 이미지를 가리키는 `@file` 인자를 제공하면, `processFileArguments`가 로딩, 크기 조정, attachment 생성을 처리한다 [packages/coding-agent/src/cli/file-processor.ts:24-50]().

- `detectSupportedImageMimeTypeFromFile`을 사용해 MIME type을 감지한다 [packages/coding-agent/src/cli/file-processor.ts:48]().
- `autoResizeImages`가 활성화되어 있으면 자동으로 `resizeImage`를 트리거한다 [packages/coding-agent/src/cli/file-processor.ts:57-58]().
- LLM 컨텍스트를 위해 dimension note가 포함된 텍스트 참조를 생성한다 [packages/coding-agent/src/cli/file-processor.ts:80-84]().
