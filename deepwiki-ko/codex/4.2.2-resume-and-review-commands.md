이 페이지는 `codex exec` 하위 시스템 안의 비대화형 `resume` 및 `review` commands의 기술적 구현을 문서화합니다. 이 command들은 headless session 복원과 sub-agent에 대한 특수 task delegation을 지원합니다.

---

## 개요

`codex exec` mode는 표준 one-shot prompt 실행 외에 두 가지 주요 특수 workflow를 제공합니다.

- **Resume**: `ThreadStore` 또는 `StateDbHandle`에서 이전에 persist된 session state를 복원합니다. [codex-rs/core/src/session/mod.rs:144-146]()
- **Review**: `ReviewTask`를 통해 finding용 structured output을 활용하여 code analysis를 수행하는 제한된 sub-agent를 spawn합니다. [codex-rs/core/src/tasks/review.rs:34-45]()

---

## Resume Command 구현

### Session Resolution과 Replay
resumption은 `Session` 및 `CodexThread` component가 관리합니다. session이 resume되면 시스템은 `ThreadStore`를 사용해 historical context를 복구합니다. [codex-rs/core/src/session/mod.rs:142-146]()

resumption process에는 다음이 포함됩니다.
1. **Event Replay**: `RolloutRecorder`와 `state_db`가 conversation history를 재구성하기 위한 historical event sequence를 제공합니다. [codex-rs/core/src/session/mod.rs:132-132](), [codex-rs/core/src/session/tests.rs:64-64]()
2. **State Restoration**: `ResumeThreadParams`는 configuration, permissions, environment selection을 복원하며 `LiveThread`를 초기화하는 데 사용됩니다. [codex-rs/core/src/session/mod.rs:141-144]()
3. **Context Injection**: resume 이후 첫 turn 동안 `run_pre_sampling_compact`는 history가 model limit 안에 있도록 보장하며, 필요하면 compaction task를 트리거합니다. [codex-rs/core/src/session/turn.rs:149-155]()

## Review Command 아키텍처

### Sub-Agent Delegation
`review` command는 `SessionTask` trait를 구현하는 `ReviewTask`를 활용합니다. [codex-rs/core/src/tasks/review.rs:42-45]() `run_codex_thread_one_shot`을 통해 새 one-shot thread를 spawn하여 execution을 sub-agent에 delegate합니다. [codex-rs/core/src/tasks/review.rs:124-135]()

review sub-agent의 주요 아키텍처 특징:
- **Capability Restriction**: sub-agent의 configuration에서 `Collab`, `WebSearchRequest`, `Goals` 같은 위험한 feature가 제거됩니다. [codex-rs/core/src/tasks/review.rs:105-113](), [codex-rs/core/src/session/review.rs:24-28]()
- **Instruction Injection**: agent에는 base instruction으로 특수 `REVIEW_PROMPT`가 제공됩니다. [codex-rs/core/src/tasks/review.rs:116-116]()
- **Structured Output**: sub-agent의 output은 `parse_review_output_event`를 사용해 `ReviewOutputEvent`로 parsing되며, model text response에서 JSON findings 추출을 시도합니다. [codex-rs/core/src/tasks/review.rs:167-173](), [codex-rs/core/src/tasks/review.rs:195-200]()

### Review Task Execution Loop
`ReviewTask::run` method는 lifecycle을 조율합니다.
1. **Start Conversation**: `start_review_conversation`을 호출해 review rubric과 input으로 sub-agent를 초기화합니다. [codex-rs/core/src/tasks/review.rs:73-79]()
2. **Event Processing**: `process_review_events`는 sub-agent의 event stream을 filtering하여 raw assistant message를 억제하고 최종 structured review output을 우선합니다. [codex-rs/core/src/tasks/review.rs:141-165]()
3. **Exit/Abort**: `exit_review_mode`를 호출해 mode를 finalize하며, 최종 findings 또는 interruption message를 렌더링합니다. [codex-rs/core/src/tasks/review.rs:84-86](), [codex-rs/core/src/tasks/review.rs:90-92]()

## 구현 세부 사항

### Thread Settings Overrides
resumption 중 또는 turn 시작 시 `CodexThreadSettingsOverrides`는 session의 동적 수정을 허용합니다. [codex-rs/core/src/codex_thread.rs:136-152]() 이러한 override는 `thread_settings_update`에서 처리되고 `SessionConfiguration`에 적용됩니다. [codex-rs/core/src/session/handlers.rs:107-156]()

### Task Lifecycle
`exec` mode의 모든 command execution은 결국 `SessionTask`를 spawn합니다. `Session`은 한 번에 하나의 task만 실행되도록 보장합니다. [codex-rs/core/src/session/session.rs:23-23]() 새 command나 interruption이 도착하면 `ActiveTurn`은 해당 `cancellation_token`을 통해 abort될 수 있습니다. [codex-rs/core/src/state/turn.rs:76-77]()
