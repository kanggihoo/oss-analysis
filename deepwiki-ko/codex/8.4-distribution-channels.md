## 목적과 범위

이 문서는 릴리스 파이프라인에서 빌드된 뒤 Codex 바이너리와 패키지가 최종 사용자에게 배포되는 방식을 설명합니다. 주요 배포 채널(GitHub Releases, npm Registry, Homebrew Cask, WinGet, DotSlash), artifact staging 및 publishing 방식, 사용자가 이용할 수 있는 설치 방법을 다룹니다.

배포 시스템은 Linux(musl), macOS(Darwin), Windows(MSVC) 전반에서 multi-platform native binary(Rust 기반)와 이에 대응하는 wrapper(Node.js/TypeScript)를 처리하도록 설계되었습니다.

## GitHub Releases

GitHub Releases는 직접 바이너리 다운로드를 위한 기본 배포 채널이며 다른 package manager의 source of truth 역할을 합니다. 릴리스 프로세스는 `rust-v*.*.*`와 일치하는 tag로 트리거됩니다 `[.github/workflows/rust-release.yml:12-15]()`.

### Artifact 구성
Artifact는 target triple별로 구성되며 Linux/macOS는 `.tar.gz` archive로, Windows는 `.zip`으로 압축됩니다.

| Target Triple | Platform | Artifact Bundle | 포함된 바이너리 |
|---------------|----------|-----------------|-------------------|
| `x86_64-unknown-linux-musl` | Linux x64 | `primary` | `codex`, `codex-responses-api-proxy`, `bwrap` |
| `aarch64-unknown-linux-musl` | Linux ARM64 | `primary` | `codex`, `codex-responses-api-proxy`, `bwrap` |
| `x86_64-apple-darwin` | macOS Intel | `primary` | `codex`, `codex-responses-api-proxy` |
| `aarch64-apple-darwin` | macOS Silicon | `primary` | `codex`, `codex-responses-api-proxy` |
| `x86_64-pc-windows-msvc` | Windows x64 | `primary` | `codex.exe`, `codex-responses-api-proxy.exe` |

릴리스 workflow는 `primary`(CLI/Proxy), `app-server`, Windows 전용 `helpers`를 포함한 여러 bundle을 빌드합니다 `[.github/workflows/rust-release.yml:77-127]()`, `[.github/workflows/rust-release-windows.yml:25-68]()`.

**출처:** `[.github/workflows/rust-release.yml:77-127]()`, `[.github/workflows/rust-release-windows.yml:25-68]()`, `[README.md:45-54]()`

## npm Registry

npm 배포는 기본 wrapper package(`@openai/codex`)와 native binary를 포함하는 여러 platform-specific optional dependency로 구성됩니다.

### npm Package 구조
`@openai/codex` 패키지는 selector 역할을 합니다. 설치되면 shim을 사용해 사용자의 현재 architecture에 맞는 native binary를 실행합니다.

```mermaid
classDiagram
    class "@openai/codex" {
        +String bin "bin/codex.js"
        +Object optionalDependencies
    }
    class "@openai/codex-linux-x64" {
        +Directory vendor/
    }
    class "codex-binary" {
        +File x86_64-unknown-linux-musl/bin/codex
    }

    "@openai/codex" ..> "@openai/codex-linux-x64" : "resolves at install"
    "@openai/codex-linux-x64" *-- "codex-binary" : "contains"
    "@openai/codex" ..> "codex-binary" : "spawns via bin/codex.js"
```

**출처:** `[codex-cli/bin/codex.js:15-22]()`, `[codex-cli/package.json:2-8]()`, `[.github/workflows/ci.yml:58-64]()`

### 실행 Shim(`bin/codex.js`)
`codex.js` 진입점은 `node_modules` tree 안에서 native binary를 찾기 위해 platform과 architecture를 런타임에 감지합니다.

1. **Platform Detection:** `process.platform`과 `process.arch`를 사용해 `PLATFORM_PACKAGE_BY_TARGET` map을 통해 target triple에 매핑합니다 `[codex-cli/bin/codex.js:15-67]()`.
2. **Path Resolution:** 로컬 `vendor` 디렉터리(개발용) 또는 `require.resolve`로 해석된 설치된 platform-specific package 안에서 바이너리를 찾습니다 `[codex-cli/bin/codex.js:78-105]()`.
3. **Signal Forwarding:** `node:child_process.spawn`을 사용해 바이너리를 실행하고, 정상 종료를 보장하기 위해 `SIGINT`, `SIGTERM`, `SIGHUP`을 자식 프로세스에 전달합니다 `[codex-cli/bin/codex.js:150-181]()`.
4. **Environment Injection:** `detectPackageManager`를 통해 감지된 package manager에 따라 `CODEX_MANAGED_BY_NPM` 또는 `CODEX_MANAGED_BY_BUN`을 설정합니다 `[codex-cli/bin/codex.js:140-148]()`.

**출처:** `[codex-cli/bin/codex.js:15-67]()`, `[codex-cli/bin/codex.js:78-105]()`, `[codex-cli/bin/codex.js:150-181]()`

### Staging과 패키징
`scripts/stage_npm_packages.py` 스크립트는 npm artifact 조립을 자동화합니다. staging 과정은 성공한 릴리스 run에서 native component를 가져와 npm 배포에 필요한 package layout을 채웁니다 `[.github/workflows/ci.yml:51-64]()`.

Stage되는 주요 native component는 다음과 같습니다.
- `codex`: 기본 CLI binary.
- `codex-responses-api-proxy`: API proxy helper.

**출처:** `[.github/workflows/ci.yml:51-64]()`, `[scripts/stage_npm_packages.py:58-62]()`

## Windows 배포와 서명

Windows 바이너리는 OS에서 신뢰되는 것으로 인식되도록 Azure Trusted Signing을 통한 code signing을 포함해 특수 처리가 필요합니다 `[.github/workflows/rust-release-windows.yml:145-156]()`.

Windows 릴리스 workflow는 다음을 빌드합니다.
- `primary`: `codex.exe`, `codex-responses-api-proxy.exe` `[.github/workflows/rust-release-windows.yml:27-33]()`.
- `helpers`: `codex-windows-sandbox-setup.exe`, `codex-command-runner.exe` `[.github/workflows/rust-release-windows.yml:41-47]()`.
- `app-server`: `codex-app-server.exe` `[.github/workflows/rust-release-windows.yml:55-61]()`.

**출처:** `[.github/workflows/rust-release-windows.yml:27-68]()`, `[.github/workflows/rust-release-windows.yml:117-144]()`
