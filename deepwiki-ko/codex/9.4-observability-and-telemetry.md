이 페이지는 Codex의 observability 인프라를 문서화하며, 주로 OpenTelemetry(OTEL) 통합을 위한 `codex-otel` crate와 usage event tracking을 위한 `codex-analytics` crate에 구현되어 있습니다. 이 시스템은 metrics, distributed tracing, structured session telemetry를 위한 통합 인터페이스를 제공합니다.

## 개요

Codex는 agent performance, tool execution, model interaction을 모니터링하기 위해 OpenTelemetry를 사용합니다. telemetry 시스템은 long-running session과 ephemeral CLI execution을 모두 처리하도록 설계되었으며, OTLP(HTTP/gRPC)와 internal analytics provider를 포함한 여러 export backend를 지원합니다.

### 데이터 흐름과 아키텍처

다음 다이어그램은 telemetry data가 core component에서 `OtelProvider`를 거쳐 OpenTelemetry SDK와 구성된 exporter로 흐르는 방식을 보여줍니다.

## codex-otel Crate

`codex-otel` crate는 OTEL SDK component의 중앙 registry 역할을 하는 `OtelProvider`를 제공합니다.

### OtelProvider와 초기화
`OtelProvider`는 OpenTelemetry `SdkLoggerProvider`, `SdkTracerProvider`, internal `MetricsClient`를 감쌉니다 `[codex-rs/otel/src/provider.rs:56-61]()`. 일반적으로 `codex-core`의 `build_provider` 함수를 사용해 초기화됩니다.

주요 함수:
* `build_provider`: application `Config`를 `OtelSettings`에 매핑하고 provider를 구성합니다. `OtelExporterKind`(예: `OtlpHttp`, `OtlpGrpc`, `Statsig`)를 internal `OtelExporter` type으로 매핑하는 일을 처리합니다 `[codex-rs/core/src/otel_init.rs:16-95]()`.
* `codex_export_filter`: `codex_otel`을 target으로 하는 event만 export되도록 보장하여 third-party library의 noise를 방지하는 filter predicate입니다 `[codex-rs/core/src/otel_init.rs:97-101]()`.
* `record_process_start`: provider가 준비되면 최초 `codex.process.start` metric을 기록하는 utility입니다 `[codex-rs/core/src/otel_init.rs:103-108]()`.
* `install_sqlite_telemetry`: `codex-rollout` 및 `codex-state` database operation을 OTEL metrics client에 연결합니다 `[codex-rs/core/src/otel_init.rs:110-117]()`.

### Export Routing 정책
Codex는 telemetry를 위한 특정 routing policy를 구현합니다. `OtelExporter` enum은 `None`, `Statsig`, `OtlpGrpc`, `OtlpHttp`를 지원합니다 `[codex-rs/otel/src/config.rs:73-90]()`.

* **Traces**: `SdkTracerProvider`를 통해 export됩니다. trace state propagation은 process-global이며 `TraceContextPropagator`를 통해 관리됩니다 `[codex-rs/otel/src/provider.rs:136-139]()`.
* **Metrics**: `MetricsClient`가 관리합니다. exporter가 `Statsig`이면 `StatsigMetricsSettings`가 global로 설치됩니다 `[codex-rs/otel/src/provider.rs:142-147]()`.
* **Logs**: `tracing` event를 OTEL log pipeline에 연결하는 `OpenTelemetryTracingBridge`를 통해 export됩니다 `[codex-rs/otel/src/provider.rs:156-165]()`.

출처: `[codex-rs/core/src/otel_init.rs:16-117]()`, `[codex-rs/otel/src/provider.rs:56-154]()`, `[codex-rs/otel/src/config.rs:73-90]()`

## Session Telemetry

`SessionTelemetry` struct는 session-scoped metadata를 관리하고 기록되는 모든 metric과 event에 올바른 tag가 붙도록 보장합니다.

### Metadata Tagging
`SessionTelemetry`가 telemetry를 기록할 때 `SessionTelemetryMetadata` 및 `AuthEnvTelemetryMetadata` 같은 metadata를 활용합니다 `[codex-rs/otel/src/lib.rs:20-22]()`. authentication mode는 `codex-core`와의 circular dependency를 피하기 위해 `AuthMode`에서 `TelemetryAuthMode`로 매핑됩니다 `[codex-rs/otel/src/lib.rs:44-63]()`.

### Turn Metadata와 Header
outbound model request의 경우 Codex는 `x-codex-turn-metadata` header에 structured metadata를 주입합니다. 이는 `TurnMetadataBag`이 관리합니다 `[codex-rs/core/src/turn_metadata.rs:127-148]()`.

* **Request Kinds**: Request는 `Turn`, `Prewarm`, `Compaction`, `Memory`로 분류됩니다 `[codex-rs/core/src/turn_metadata.rs:78-83]()`.
* **Lineage**: header에는 `session_id`, `thread_id`, 그리고 `forked_from_thread_id`, `parent_thread_id` 같은 lineage field가 포함됩니다 `[codex-rs/core/src/turn_metadata.rs:131-137]()`.
* **Workspace Info**: Git metadata(remote URL, commit hash, dirty status)가 workspace root별로 수집되어 첨부됩니다 `[codex-rs/core/src/turn_metadata.rs:86-108]()`.

### Runtime Metrics Summary
`RuntimeMetricsSummary` struct는 특정 execution window(일반적으로 단일 turn 또는 session)에 대한 counter와 histogram을 집계합니다 `[codex-rs/otel/src/metrics/runtime_metrics.rs:10-30]()` . 이 구조체는 다음을 추적합니다.
* Tool call count와 duration `[codex-rs/otel/src/metrics/runtime_metrics.rs:11-12]()`.
* inference engine이 보고하는 TTFT 및 TBT(Time Between Tokens)를 포함한 API request metrics `[codex-rs/otel/src/metrics/runtime_metrics.rs:13-25]()`.
* Streaming event count(SSE 및 WebSocket) `[codex-rs/otel/src/metrics/runtime_metrics.rs:17-22]()`.

## Analytics Crate(Usage Tracking)

`codex-analytics` crate는 고수준 product usage event를 담당합니다. 이 crate는 internal system fact를 structured event request로 변환합니다.

### AnalyticsReducer와 Client
`AnalyticsEventsClient`는 internal `AnalyticsEventsQueue`를 관리합니다 `[codex-rs/analytics/src/client.rs:56-58]()`. background task는 `AnalyticsReducer`를 실행해 `AnalyticsFact` 객체를 수집하고 `TrackEventRequest` batch를 내보냅니다 `[codex-rs/analytics/src/client.rs:61-70]()`.

**Analytics Event Generation**

출처: `[codex-rs/analytics/src/reducer.rs:132-140]()`, `[codex-rs/analytics/src/events.rs:58-83]()`

### 주요 Event 범주
* **Thread Initialization**: `ThreadInitializedEvent`를 통해 thread가 시작되는 방식(`New`, `Forked`, `Resumed`)을 추적합니다 `[codex-rs/analytics/src/events.rs:164-167]()`.
* **Guardian Reviews**: `GuardianReviewDecision`(Approved, Denied, Aborted)과 failure reason을 포함해 security decision을 상세히 추적합니다 `[codex-rs/analytics/src/events.rs:170-191]()`.
* **History Compaction**: `CodexCompactionEventRequest`를 통해 compaction system의 성능과 trigger를 모니터링합니다 `[codex-rs/analytics/src/events.rs:65]()`. compaction status는 `CompactionStatus`(Success, Failed, Interrupted)를 사용해 추적됩니다 `[codex-rs/analytics/src/facts.rs:50]()`.
* **Skill Invocations**: 어떤 skill이 사용되었고 명시적 또는 암묵적으로 호출되었는지 추적합니다 `[codex-rs/analytics/src/events.rs:112-129]()`.
* **Fingerprinting**: attribution과 quality metrics를 위해 `CodexAcceptedLineFingerprintsEventRequest`를 통해 수락된 code change를 추적합니다 `[codex-rs/analytics/src/events.rs:106-109]()`.
* **Plugin Telemetry**: installation, uninstallation, usage를 포함한 plugin lifecycle event를 추적합니다 `[codex-rs/analytics/src/events.rs:78-82]()`.

### 구현 세부 사항
`AnalyticsReducer`는 completion event와 start time의 상관관계를 파악하기 위해 진행 중인 request와 turn의 state를 유지합니다 `[codex-rs/analytics/src/reducer.rs:132-140]()`. 단일 turn 내 over-reporting을 피하기 위해 `AppUsed`, `PluginUsed` 같은 event에 대한 복잡한 deduplication을 처리합니다 `[codex-rs/analytics/src/client.rs:85-117]()`.

오류는 세밀한 failure analysis를 위해 `CodexErrKind`로 분류되며, `ContextWindowExceeded`, `UsageLimitReached`, `CyberPolicy` violation 같은 reason을 캡처합니다 `[codex-rs/analytics/src/facts.rs:140-177]()`.
