## Docker Build Architecture

### Multi-Stage Build Pipeline

LangfuseлҠ” `pnpm` л°Ҹ `turbo` к°ҷмқҖ build toolмқҙ final runtime imageм—җ нҸ¬н•Ёлҗҳм§Җ м•ҠлҸ„лЎқ sophisticated multi-stage pipelineмқ„ мӮ¬мҡ©н•©лӢҲлӢӨ. Build processлҠ” `turbo prune`мқ„ нҷңмҡ©н•ҙ к°Ғ serviceм—җ лҢҖн•ң monorepo subsetмқ„ мғқм„ұн•©лӢҲлӢӨ.

**Build Stages Description**

1.  **pruner**: `turbo prune --scope=<service> --docker`лҘј мӢӨн–үн•©лӢҲлӢӨ. мқҙ commandлҠ” нҠ№м • service(web лҳҗлҠ” worker)мҷҖ `@langfuse/shared` к°ҷмқҖ лӮҙл¶Җ workspace dependencyм—җ н•„мҡ”н•ң codeмҷҖ `package.json` fileл§Ң м¶”м¶ңн•©лӢҲлӢӨ [web/Dockerfile:37-42](), [worker/Dockerfile:22-27]().
2.  **builder**: `pnpm install --frozen-lockfile`мқ„ мҲҳн–үн•©лӢҲлӢӨ. лҳҗн•ң `NEXT_PUBLIC_POSTHOG_KEY`мІҳлҹј Next.js frontend bundleм—җ н•„мҡ”н•ң build-time argument(`ARG`)лҘј environment variable(`ENV`)м—җ bakeн•©лӢҲлӢӨ [web/Dockerfile:44-79]().
3.  **migrate-builder**: `golang:1.26`мқ„ мӮ¬мҡ©н•ҳлҠ” Go-based stageлЎң, prebuilt binaryмҷҖ кҙҖл Ёлҗң CVEлҘј н”јн•ҳкё° мң„н•ҙ ClickHouse supportк°Җ нҸ¬н•Ёлҗң `migrate` CLIлҘј compileн•©лӢҲлӢӨ [web/Dockerfile:23-35]().
4.  **runner**: Final production imageмһ…лӢҲлӢӨ. Webмқҳ standalone output лҳҗлҠ” Workerмқҳ compiled `dist` folderл§Ң copyн•ҳкі , `prisma` к°ҷмқҖ runtime-only toolмқ„ installн•ң л’Ө non-root userлЎң м „нҷҳн•©лӢҲлӢӨ [web/Dockerfile:112-170](), [worker/Dockerfile:68-92]().

м¶ңмІҳ: [web/Dockerfile:1-176](), [worker/Dockerfile:1-101](), [package.json:100](), [web/Dockerfile:10](), [worker/Dockerfile:10](), [package.json:51]()

## Container Orchestration л°Ҹ Configuration

### Docker Compose
Self-hostingмқ„ мң„н•ң primary orchestration methodлҠ” Docker Composeмһ…лӢҲлӢӨ. Standard stackм—җлҠ” лӢӨмқҢмқҙ нҸ¬н•Ёлҗ©лӢҲлӢӨ.
- `langfuse-web`: Main application(Next.js) [docker-compose.yml:71]().
- `langfuse-worker`: Background processor(Express/BullMQ) [docker-compose.yml:7]().
- `postgres`: Metadata storage [docker-compose.yml:147]().
- `clickhouse`: Observability data storage [docker-compose.yml:90]().
- `redis`: Queue backend(BullMQ) л°Ҹ caching [docker-compose.yml:132]().
- `minio`: Event bodyмҷҖ exportлҘј мң„н•ң S3-compatible blob storage [docker-compose.yml:111]().

## Deployment Initialization Flow

Langfuse containerк°Җ мӢңмһ‘лҗ  л•Ң, entrypoint scriptм—җ м •мқҳлҗң нҠ№м • sequenceлҘј л”°лқј environmentк°Җ мӨҖл№„лҗҳм—ҲлҠ”м§Җ ліҙмһҘн•©лӢҲлӢӨ.

```mermaid
sequenceDiagram
    participant OS as "Container OS"
    participant EP as "entrypoint.sh"
    participant PG as "PostgreSQL"
    participant CH as "ClickHouse"
    participant APP as "Node.js Process"

    OS->>EP: "Execute Entrypoint (dumb-init)"
    Note over EP: "web/entrypoint.sh"
    EP->>EP: "Check/Construct DATABASE_URL"
    EP->>PG: "prisma migrate deploy"
    EP->>CH: "migrate -path ... up"
    EP->>APP: "exec node web/server.js"
```

**Key Implementation Details:**
- **PostgreSQL Migrations**: Schemaк°Җ мөңмӢ  мғҒнғңмқём§Җ ліҙмһҘн•ҳкё° мң„н•ҙ web container lifecycle мӨ‘ `prisma migrate deploy`лҘј нҶөн•ҙ мӢӨн–үлҗ©лӢҲлӢӨ [web/Dockerfile:177]().
- **ClickHouse Migrations**: `migrate-builder` stageм—җм„ң compileлҗҳм–ҙ runnerлЎң copyлҗҳлҠ” `migrate` CLIлҘј нҶөн•ҙ кҙҖлҰ¬лҗ©лӢҲлӢӨ [web/Dockerfile:152]().
- **Signal Handling**: ContainerлҠ” signalмқ„ мҳ¬л°”лҘҙкІҢ мІҳлҰ¬н•ҳкё° мң„н•ҙ `dumb-init`мқ„ мӮ¬мҡ©н•ҳкі , Next.jsк°Җ graceful shutdownмқ„ мІҳлҰ¬н•  мҲҳ мһҲлҸ„лЎқ `NEXT_MANUAL_SIG_HANDLE=true`лҘј мӮ¬мҡ©н•©лӢҲлӢӨ [web/Dockerfile:177](), [web/package.json:19]().
- **Cleanup**: Database maintenance taskлҘј мң„н•ң `cleanup.sql` scriptк°Җ м ңкіөлҗ©лӢҲлӢӨ [packages/shared/package.json:10](), [web/Dockerfile:167]().

м¶ңмІҳ: [web/Dockerfile:152-177](), [worker/Dockerfile:97-102](), [web/package.json:19](), [packages/shared/package.json:10-11]()
