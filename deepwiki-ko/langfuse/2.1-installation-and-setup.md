## 목적 및 범위

이 문서는 Langfuse monorepo의 로컬 개발 환경을 설정하기 위한 기술 지침을 제공합니다. prerequisite, `pnpm`을 사용한 dependency management, Docker를 통한 infrastructure orchestration, 이중 database 아키텍처(PostgreSQL 및 ClickHouse)의 초기화를 다룹니다.

## 시스템 요구 사항

### Node.js Runtime
Langfuse는 **Node.js version 24**를 요구합니다. 이는 서비스에서 사용되는 Next.js 및 최신 OpenTelemetry instrumentation과의 호환성을 보장하기 위해 monorepo configuration 전반에서 강제됩니다.

| Component | Requirement | Source |
| :--- | :--- | :--- |
| Node Version | `24` | [package.json:8-8](), [web/package.json:7-7](), [worker/package.json:12-12]() |
| Web Engine | `node: 24` | [web/package.json:7-7]() |
| Worker Engine | `node: 24` | [worker/package.json:12-12]() |

### Package Manager
프로젝트는 **pnpm version 11.1.3**을 사용합니다. lockfile 무결성과 효율적인 workspace linking을 유지하기 위해 `preinstall` script를 통해 `pnpm` 사용이 엄격히 강제됩니다.

```bash
# Corepack is used to manage the pinned version
corepack enable
corepack prepare pnpm@11.1.3 --activate
```

**출처:** [package.json:14-14](), [package.json:100-100](), [CONTRIBUTING.md:114-114]()

### 핵심 기술 스택
개발 환경은 다음 핵심 tool에 의존합니다.
*   **Turbo**: monorepo 전반의 task(build, lint, test)를 orchestrate합니다 [package.json:51-51]().
*   **Docker**: `docker-compose.dev.yml`에 정의된 local infrastructure stack을 실행하는 데 필요합니다 [package.json:15-17]().
*   **ClickHouse Client**: 수동 database inspection 및 migration script 실행에 필요합니다 [CONTRIBUTING.md:116-116]().
*   **golang-migrate**: custom-built CLI를 통해 ClickHouse database migration을 관리하는 데 사용됩니다 [web/Dockerfile:23-35]().

### Installation Command
모든 dependency를 설치하고 내부 workspace package를 link하려면 다음을 실행하세요.
```bash
pnpm install
```
repository는 repository-specific setup task를 위한 helper bash script를 실행하는 `postinstall` script를 사용합니다 [package.json:13-13]().

## Infrastructure Setup

### Local Services
Langfuse에는 네 가지 주요 infrastructure component가 필요합니다. 이들은 `docker-compose.dev.yml`을 통해 관리됩니다.

| Service | Technology | Usage |
| :--- | :--- | :--- |
| **Database** | PostgreSQL | Transactional data(User, Project, API Key) [CONTRIBUTING.md:70-70]() |
| **Analytics** | ClickHouse | OLAP data(Trace, Observation, Score) [CONTRIBUTING.md:72-72]() |
| **Cache/Queue** | Redis | BullMQ job coordination 및 caching [CONTRIBUTING.md:71-71]() |
| **Storage** | S3 / Blob | Raw event storage 및 multi-modal attachment [CONTRIBUTING.md:73-73]() |

**출처:** [CONTRIBUTING.md:64-89](), [package.json:15-17]()

### Management Commands
Infrastructure는 root-level script를 통해 제어됩니다.
*   `pnpm run infra:dev:up`: local Docker stack을 시작하고 health check를 기다립니다 [package.json:15-15]().
*   `pnpm run infra:dev:down`: local Docker stack을 중지합니다 [package.json:16-16]().
*   `pnpm run infra:dev:prune`: stack을 중지하고 모든 volume을 제거합니다(data reset) [package.json:17-17]().

## Database Initialization

### PostgreSQL & Prisma
PostgreSQL 초기화에는 Prisma client 생성과 migration 적용이 포함됩니다. 이 task들은 `turbo`를 통해 orchestrate됩니다.

```bash
# Generate Prisma Client
pnpm run db:generate

# Apply migrations to local Postgres
pnpm run db:migrate

# Seed initial development data
pnpm run db:seed
```

**출처:** [package.json:18-20](), [packages/shared/package.json:67-73]()

### ClickHouse Migrations
ClickHouse schema management는 shell script와 전용 seeder의 조합을 사용해 처리됩니다.

*   `ch:up`: `clickhouse/scripts/up.sh`를 사용해 ClickHouse container를 시작합니다 [packages/shared/package.json:76-76]().
*   `ch:dev-tables`: development-specific table structure를 초기화하기 위해 `dev-tables.sh`를 실행합니다 [packages/shared/package.json:79-79]().
*   `ch:seed`: OLAP database를 채우기 위해 `seed-clickhouse.ts`를 실행합니다 [packages/shared/package.json:81-81]().

## 간소화된 Onboarding: `dx` Command

"one-command" setup을 위해 Langfuse는 `dx` script를 제공합니다. 이는 신규 contributor가 example data를 포함해 완전히 동작하는 환경을 얻기 위한 권장 경로입니다.

```bash
pnpm run dx
```

**`dx`가 수행하는 작업 순서:**
1.  **Install**: dependency 설치를 위해 `pnpm i`를 실행합니다 [package.json:23-23]().
2.  **Reset Infrastructure**: 기존 상태를 지우기 위해 `infra:dev:prune`을 실행합니다 [package.json:23-23]().
3.  **Start Services**: 새 container를 가져오고 시작하기 위해 `infra:dev:up --pull always`를 실행합니다 [package.json:23-23]().
4.  **Postgres Reset**: `db:reset:test` 및 `db:reset`을 통해 transactional database를 reset합니다 [package.json:23-23]().
5.  **ClickHouse Reset**: `ch:down`, `ch:up`, `ch:seed`, `ch:dev-tables`를 처리하는 `ch:reset`을 통해 OLAP database를 reset합니다 [package.json:23-23](), [packages/shared/package.json:80-80]().
6.  **Seed Examples**: `db:seed:examples`를 통해 example data로 환경을 채웁니다 [package.json:23-23]().
7.  **Launch**: `pnpm run dev`를 통해 `web` 및 `worker`용 development server를 시작합니다 [package.json:23-23]().

**출처:** [package.json:23-25](), [CONTRIBUTING.md:154-155](), [packages/shared/package.json:80-80]()
