이 페이지는 대규모 dataset을 처리하기 위해 Langfuse 웹 애플리케이션에서 사용하는 성능 최적화 기법을 문서화하며, 특히 복잡한 JSON 및 Markdown 구조를 위한 virtualization, progressive data loading, efficient rendering strategy에 초점을 둡니다.

## 개요

Langfuse는 수백만 개의 trace, observation, score를 관리합니다. responsive UI를 유지하기 위해 frontend는 여러 optimization layer를 사용합니다.

- **List & Tree Virtualization**: `@tanstack/react-virtual`과 최적화된 table row expansion을 사용해 viewport에 현재 보이는 element만 rendering합니다.
- **Progressive Loading**: lightweight metadata와 heavy analytical metrics 및 큰 I/O field를 별도의 processing step으로 분리합니다.
- **Deep JSON Parsing**: JSON object 안에 중첩된 stringified JSON을 처리하기 위해 `@langfuse/shared`의 `deepParseJson`을 사용합니다 [web/src/components/ui/CodeJsonViewer.tsx:48-48]().
- **Smart Expansion**: DOM bloat를 방지하기 위해 table view에서 nested object expansion depth를 제어합니다 [web/src/components/ui/PrettyJsonView.tsx:71-75]().
- **Unicode Decoding**: 큰 payload에서 browser hang을 방지하기 위해 depth와 node-count cap을 두고 `\uXXXX` escape sequence를 iterative하게 decode합니다 [web/src/components/ui/PrettyJsonView.tsx:89-103]().

## Virtualization & Table Architecture

### Virtualizer and Table Implementation

Langfuse는 Input/Output(I/O) data에 대해 lightweight preview에서 full rich-text view로 이동하는 tiered rendering strategy를 사용합니다.

## Performance Optimization Strategies

### Unicode Decoding Performance
Python의 `json.dumps(ensure_ascii=True)`를 통해 ingest된 trace에는 double-escaped non-ASCII character가 포함되는 경우가 많습니다. `decodeUnicodeInJson`은 이를 iterative하게 처리합니다.
- **Node Cap**: `DECODE_UNICODE_MAX_NODES = 50,000` 이후 processing을 중단합니다 [web/src/components/ui/PrettyJsonView.tsx:89-89]().
- **Depth Cap**: stack overflow를 방지하기 위해 recursion을 `DECODE_UNICODE_MAX_DEPTH = 200`으로 제한합니다 [web/src/components/ui/PrettyJsonView.tsx:90-90]().
- **Explicit Stack**: browser responsiveness를 유지하기 위해 recursion 대신 iterative approach를 사용합니다 [web/src/components/ui/PrettyJsonView.tsx:132-134]().

### Large Document Handling
- **Markdown AST Transformation**: `MarkdownViewer`는 전체 document를 다시 parsing하지 않고 text node를 split하고 `PromptReferenceButton` component를 inject하기 위해 custom `remarkPromptReferences` plugin을 사용합니다 [web/src/components/ui/MarkdownViewer.tsx:203-223]().
- **Table Data Management**: Core data와 metrics는 종종 별도로 fetch된 뒤 `joinTableCoreAndMetrics`를 사용해 UI에서 join됩니다. 이를 통해 expensive metric aggregation을 기다리는 동안 table이 metadata를 render할 수 있습니다 [web/src/components/table/use-cases/traces.tsx:54-54](), [web/src/features/prompts/components/prompts-table.tsx:23-23]().
- **Memoization**: `TracesTable`, `ObservationsTable`, `SessionsTable` 같은 table은 매 render마다 expensive recalculation이 발생하지 않도록 column definition과 filter configuration에 `useMemo`를 사용합니다 [web/src/components/table/use-cases/traces.tsx:166-169](), [web/src/components/table/use-cases/observations.tsx:165-168](), [web/src/components/table/use-cases/sessions.tsx:99-102]().
