GNOME 확장은 GNOME Shell JavaScript 환경(GJS)과 CodeBurn CLI 사이를 연결하기 위해 특화된 data client를 사용합니다. 이 페이지는 `DataClient` 구현, GSettings schema 구성, Linux 환경의 설치 생명주기를 문서화합니다.

## DataClient 구현

`DataClient` 클래스는 `codeburn` CLI를 subprocess로 실행하고, JSON 출력을 파싱하며, 결과의 인메모리 캐시를 관리하는 역할을 담당합니다. 비동기 프로세스 관리를 위해 `Gio`와 `GLib` 라이브러리를 활용합니다.

### 주요 기능
*   **Subprocess 관리**: 사용자 정의 `PATH`로 CLI 명령을 실행하기 위해 `Gio.SubprocessLauncher`를 사용합니다 [gnome/dataClient.js:111-115]().
*   **요청 취소**: 빠른 기간 전환 중 race condition을 방지하기 위해 `Gio.Cancellable`을 통한 진행 중 요청 취소를 지원합니다 [gnome/dataClient.js:37-42]().
*   **인자 검증**: shell injection을 방지하기 위해 `codeburn-path`를 `SAFE_ARG_RE`(`/^[A-Za-z0-9 ._/\-]+$/`)와 대조해 검증합니다 [gnome/dataClient.js:5-6, 68]().
*   **경로 확장**: 표준 시스템 `PATH`에 없는 경우 `codeburn` binary를 찾기 위해 일반적인 package manager 디렉터리(NPM, Volta, Bun, Cargo, ASDF, FNM, PNPM)를 자동으로 스캔합니다 [gnome/dataClient.js:7-20, 88-96]().

### Timeout과 오류 처리
client는 15초의 `TIMEOUT_SECONDS` 제한을 강제합니다 [gnome/dataClient.js:4](). CLI가 이 시간 안에 반환하지 않으면 `GLib.timeout_add_seconds`가 `proc.force_exit()`를 트리거하고 promise를 reject합니다 [gnome/dataClient.js:121-126]().

## 구성과 GSettings

확장의 상태와 사용자 preference는 `org.gnome.shell.extensions.codeburn.gschema.xml`에 정의된 GNOME GSettings 시스템을 사용해 영속화됩니다.

### Schema 키
| 키 | 타입 | 기본값 | 설명 |
|:---|:---|:---|:---|
| `refresh-interval` | uint | 30 | 데이터 refresh 사이의 초 단위 시간입니다(범위: 5-300) [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:6-11]() |
| `default-period` | string | 'today' | 초기 보기입니다(today, week, 30days, month, all) [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:13-17]() |
| `budget-threshold` | double | 0.0 | 일일 지출 한도(USD)입니다 [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:19-23]() |
| `budget-alert-enabled` | boolean | false | 일일 예산 경고 토글입니다 [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:25-29]() |
| `compact-mode` | boolean | false | 상단 패널에서 비용 라벨을 숨깁니다 [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:31-35]() |
| `codeburn-path` | string | '' | `codeburn` binary 위치에 대한 수동 override입니다 [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:49-53]() |
| `disabled-providers` | as (array) | [] | 계산에서 제외할 provider ID 목록입니다 [gnome/schemas/org.gnome.shell.extensions.codeburn.gschema.xml:61-65]() |

## 설치 생명주기

`install.sh` 스크립트는 확장을 로컬 사용자 디렉터리에 배포하는 과정을 자동화합니다.

### 설치 단계
1.  **Schema 컴파일**: GNOME이 요구하는 `gschemas.compiled` binary를 생성하기 위해 `schemas/` 디렉터리에서 `glib-compile-schemas`를 실행합니다 [gnome/install.sh:11-12]().
2.  **디렉터리 생성**: 대상 디렉터리 `~/.local/share/gnome-shell/extensions/codeburn@codeburn.dev`가 존재하도록 보장합니다 [gnome/install.sh:4-5, 15]().
3.  **파일 마이그레이션**: 로직(`extension.js`, `indicator.js`, `dataClient.js`), metadata, stylesheet, 컴파일된 schema를 extension 폴더로 복사합니다 [gnome/install.sh:18-31]().

### 설치 후
사용자는 GNOME Shell을 재시작하거나(Wayland에서는 로그아웃/로그인) 다음 명령으로 확장을 활성화해야 합니다.
`gnome-extensions enable codeburn@codeburn.dev` [gnome/install.sh:37]().

출처: [gnome/install.sh:1-39](), [gnome/README.md:11-27]()
