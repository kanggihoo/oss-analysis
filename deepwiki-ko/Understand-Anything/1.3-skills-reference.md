이 페이지는 Understand Anything 플러그인에서 사용할 수 있는 모든 사용자 대상 슬래시 명령(스킬)에 대한 간결한 참조를 제공합니다. 각 스킬의 목적, 인자, 플래그, 예상 출력이 자세히 설명되어 있습니다. 개발자가 포괄적으로 이해할 수 있도록 기술 구현 참고 사항, 데이터 흐름, 핵심 함수/클래스도 포함되어 있습니다.

## `/understand`

아키텍처, 구성 요소, 관계를 이해하기 위한 대화형 지식 그래프를 생성하도록 코드베이스를 분석합니다. 이는 다른 여러 스킬의 기본 데이터 구조를 생성하는 핵심 스킬입니다.

### 인자와 플래그

*   `[path]`(선택 사항): 현재 작업 디렉터리 대신 분석할 디렉터리 경로(예: `/path/to/repo` 또는 `../other-project`)입니다.
*   `--full`: 기존 그래프를 무시하고 전체 재빌드를 강제합니다.
*   `--auto-update`: commit 시 자동 그래프 업데이트를 활성화합니다(`.understand-anything/config.json`에 `autoUpdate: true`를 씁니다).
*   `--no-auto-update`: 자동 그래프 업데이트를 비활성화합니다(`.understand-anything/config.json`에 `autoUpdate: false`를 씁니다).
*   `--review`: inline deterministic validation 대신 전체 LLM graph-reviewer를 실행합니다.
*   `--language <lang>`: 모든 텍스트 콘텐츠(요약, 설명, 태그, 제목, languageNotes, languageLesson)를 지정된 언어로 생성합니다. ISO 639-1 코드(예: `zh`, `ja`, `ko`, `en`, `es`, `fr`, `de`) 또는 친숙한 이름(예: `chinese`, `japanese`, `korean`, `english`, `spanish`)을 허용합니다. `zh-TW`, `zh-HK` 같은 locale variant도 지원됩니다. 기본값은 `en`(English)입니다. 이 선호 설정은 `.understand-anything/config.json`에 저장됩니다.

### 예상 출력

주요 출력은 프로젝트 루트의 `.understand-anything/` 디렉터리에 위치한 `knowledge-graph.json` 파일입니다. 이 파일에는 코드베이스의 구조와 관계를 나타내는 nodes, edges, layers, tour steps가 포함됩니다.

### 구현 세부 사항

`/understand` 스킬은 셸 스크립트가 조율하는 다단계 프로세스입니다 [understand-anything-plugin/skills/understand/SKILL.md]().

#### Phase 0: Pre-flight

이 단계는 `PROJECT_ROOT`를 결정하고 플러그인이 빌드되어 있는지 확인합니다.
1.  **`PROJECT_ROOT` 해결**:
    *   path 인자가 제공되면 절대 경로로 해석됩니다.
    *   path가 제공되지 않으면 현재 작업 디렉터리가 사용됩니다.
    *   **Worktree Redirect**: 스크립트는 `git rev-parse --git-dir`와 `git rev-parse --git-common-dir`를 비교하여 `PROJECT_ROOT`가 Git worktree 안에 있는지 확인합니다. worktree라면 임시 그래프 데이터를 방지하기 위해 `PROJECT_ROOT`가 메인 저장소 루트로 리다이렉트됩니다 [understand-anything-plugin/skills/understand/SKILL.md:53-70](). 이 동작은 `UNDERSTAND_NO_WORKTREE_REDIRECT=1`을 설정해 재정의할 수 있습니다.
2.  **플러그인 빌드 확인**: 스크립트는 `packages/core/dist/index.js`가 존재하는지 검증합니다. 없으면 `pnpm install`과 `pnpm --filter @understand-anything/core build`를 실행해 core 라이브러리를 컴파일합니다 [understand-anything-plugin/skills/understand/SKILL.md:116-117](). `PLUGIN_ROOT`는 다양한 설치 시나리오를 처리하기 위해 일련의 후보 경로를 통해 해석됩니다 [understand-anything-plugin/skills/understand/SKILL.md:88-99]().

#### Phase 1: Project Scanning & File Discovery

이 단계는 프로젝트의 모든 관련 파일을 식별합니다.
1.  **File Enumeration**: `scan-project.mjs` 스크립트가 실행되어 파일을 탐색하고, `.understandignore` 규칙을 적용하며, 언어를 감지합니다 [understand-anything-plugin/skills/understand/SKILL.md:129-130]().
2.  **Import Map Generation**: `extract-import-map.mjs`는 Tree-sitter를 사용해 각 파일의 import map을 생성하며, 이는 파일 간 의존성을 해석하는 데 중요합니다 [understand-anything-plugin/skills/understand/SKILL.md:131-132]().

#### Phase 2: File Analysis & Batch Processing

이 단계는 파일을 batch로 처리해 구조 및 의미 정보를 추출합니다.
1.  **Batch Computation**: `compute-batches.mjs`는 토큰 제한과 파일 범주를 고려하여 발견된 파일을 batch로 나눕니다 [understand-anything-plugin/skills/understand/SKILL.md:140-141]().
2.  **Parallel Analysis**: 각 batch는 `file-analyzer` 에이전트로 dispatch됩니다. 에이전트의 작업은 소스 파일을 읽고 구조화된 지식 그래프 데이터(nodes와 edges)를 생성하는 것입니다 [understand-anything-plugin/agents/file-analyzer.md:9-11]().
    *   **Structural Extraction**: `file-analyzer`는 번들된 `extract-structure.mjs` 스크립트를 실행합니다. 이 스크립트는 코드 파일에는 Tree-sitter를, 비코드 파일에는 특화 parser를 사용해 구조 데이터를 deterministic하게 추출합니다 [understand-anything-plugin/agents/file-analyzer.md:28-30, 67-74](). 출력은 functions, classes, exports, call graphs, metrics를 포함하는 JSON 객체입니다 [understand-anything-plugin/agents/file-analyzer.md:83-117](). 비코드 파일의 경우 definitions, services, endpoints, steps, resources를 추출합니다 [understand-anything-plugin/agents/file-analyzer.md:121-128]().
    *   **LLM Semantic Analysis**: 구조 추출 후 `file-analyzer`는 전문가 판단을 적용해 summaries, tags, complexity ratings, semantic edges를 생성합니다. cross-batch edges의 신뢰도를 높이기 위해 cross-batch context로 `neighborMap`을 사용합니다 [understand-anything-plugin/agents/file-analyzer.md:55-62]().
3.  **Batch Merging**: `merge-batch-graphs.py`는 모든 batch의 결과를 하나의 `assembled-graph.json` 파일로 결합합니다. 이 스크립트는 IDs, complexity values를 정규화하고 edges를 정리합니다 [understand-anything-plugin/skills/understand/merge-batch-graphs.py:3-8](). ID canonicalization(예: `func:`에서 `function:`으로)을 처리하고 nodes와 edges를 중복 제거합니다 [understand-anything-plugin/skills/understand/merge-batch-graphs.py:178-199, 202-210]().

#### Phase 3: Graph Assembly & Validation

이 단계는 조립된 그래프를 정제하고 검증합니다.
1.  **Semantic Recovery**: `assemble-reviewer` 에이전트는 deterministic scripts가 잡아내지 못하는 의미론적 문제를 찾기 위해 `assembled-graph.json`을 검토합니다 [understand-anything-plugin/skills/understand/SKILL.md:159-160]().
2.  **Referential Integrity**: `graph-reviewer`는 그래프의 일관성을 보장하기 위한 검사를 수행합니다.

#### Phase 4: Architecture Analysis & Tour Building

이 단계는 그래프에 아키텍처 layer와 guided tour를 추가합니다.
1.  **Layer Assignment**: `architecture-analyzer` 에이전트는 `intraGroupDensity`와 `interGroupImports` 같은 휴리스틱을 기반으로 nodes를 architectural layers에 할당합니다 [understand-anything-plugin/skills/understand/SKILL.md:167-168]().
2.  **Tour Generation**: `tour-builder` 에이전트는 BFS와 fan-in ranking을 사용해 guided learning path(tour)를 생성합니다 [understand-anything-plugin/skills/understand/SKILL.md:169-170]().

#### Phase 5: Finalization

이 단계는 subdomain graphs를 병합하고 최종 지식 그래프를 저장합니다.
1.  **Subdomain Merging**: `merge-subdomain-graphs.py`는 subdomain knowledge graphs(예: `/understand-domain` 또는 `/understand-knowledge`에서 생성된 것)를 메인 `knowledge-graph.json`에 병합합니다 [understand-anything-plugin/skills/understand/merge-subdomain-graphs.py:3-8](). nodes와 edges를 중복 제거하고, ID 기준으로 layers를 병합하며, tour steps를 이어 붙입니다 [understand-anything-plugin/skills/understand/merge-subdomain-graphs.py:64-157]().
2.  **Save Graph**: 최종 `knowledge-graph.json`이 저장됩니다.

#### 진행 상황 보고

이 스킬은 단계 전환과 batch 처리 상태를 포함해 상세한 진행 상황 업데이트를 사용자에게 제공합니다 [understand-anything-plugin/skills/understand/SKILL.md:23-37]().

#### GraphBuilder 클래스

`GraphBuilder` 클래스 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:60-60]()는 지식 그래프 구성의 중심입니다. 다양한 타입의 nodes와 edges를 추가하는 메서드를 제공합니다.
*   `addFile(filePath: string, meta: FileMeta)`: file node를 추가합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:84-102]().
*   `addFileWithAnalysis(filePath: string, analysis: StructuralAnalysis, meta: FileAnalysisMeta)`: file node와 그 안에 포함된 functions 및 classes를 추가하고 `contains` edges를 생성합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:105-177]().
*   `addImportEdge(fromFile: string, toFile: string)`: 파일 간 `imports` edge를 추가합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:179-190]().
*   `addCallEdge(callerFile: string, callerFunc: string, calleeFile: string, calleeFunc: string)`: functions 간 `calls` edge를 추가합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:192-208]().
*   `addNonCodeFile(filePath: string, meta: NonCodeFileMeta)`: non-code file node를 추가합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:210-225]().
*   `addNonCodeFileWithAnalysis(filePath: string, meta: NonCodeFileAnalysisMeta)`: non-code file node와 그 child nodes(definitions, services, endpoints, steps, resources)를 추가하고 `contains` edges를 생성합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:228-270]().

`GraphBuilder`는 언어 감지도 처리하며 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:77-79](), 고유한 node 및 edge IDs를 보장합니다 [understand-anything-plugin/packages/core/src/analyzer/graph-builder.ts:64-66]().

## `/understand-dashboard`

프로젝트의 지식 그래프를 탐색하기 위한 대화형 시각화 대시보드를 실행합니다.

### 인자와 플래그

제공된 `/understand-dashboard`용 `SKILL.md`에는 명시적으로 문서화된 내용이 없습니다. 일반적으로 `/understand`가 `knowledge-graph.json`을 생성한 뒤 호출됩니다.

### 예상 출력

사용자의 브라우저에서 웹 기반 대시보드를 열어 지식 그래프를 표시합니다.

### 구현 세부 사항

대시보드는 `knowledge-graph.json` 파일을 읽는 React 기반 애플리케이션입니다. 그래프 데이터를 제공하기 위해 data API middleware가 포함된 Vite dev server를 사용합니다.

## `/understand-chat`

생성된 지식 그래프를 사용해 코드베이스에 관한 질문에 답합니다.

### 인자와 플래그

*   `[query]`: 코드베이스에 관한 자연어 질문입니다.

### 예상 출력

지식 그래프의 특정 파일, 함수, 관계, architectural layers를 참조하는 사용자 질의에 대한 자연어 답변입니다.

### 구현 세부 사항

`/understand-chat` 스킬은 `knowledge-graph.json` 파일을 질의하는 방식으로 동작합니다.
1.  **Prerequisite Check**: `.understand-anything/knowledge-graph.json`의 존재를 검증합니다. 찾을 수 없으면 사용자에게 먼저 `/understand`를 실행하라고 안내합니다 [understand-anything-plugin/skills/understand-chat/SKILL.md:34-35]().
2.  **Project Metadata**: context를 위해 그래프에서 `project` 섹션(name, description, languages, frameworks)을 추출합니다 [understand-anything-plugin/skills/understand-chat/SKILL.md:37-38]().
3.  **Node Search**: `grep`을 사용해 사용자의 query에서 얻은 keywords를 node의 `name`, `summary`, `tags` 필드에서 검색합니다 [understand-anything-plugin/skills/understand-chat/SKILL.md:41-43]().
4.  **Edge Traversal (1-hop)**: 일치하는 각 node ID에 대해 `edges` 섹션을 검색해 연결된 nodes(상류 및 하류 의존성 모두)를 찾고 1-hop subgraph를 형성합니다 [understand-anything-plugin/skills/understand-chat/SKILL.md:45-48]().
5.  **Layer Context**: `layers` 섹션을 검색해 일치한 nodes가 어떤 architectural layers에 속하는지 식별합니다 [understand-anything-plugin/skills/understand-chat/SKILL.md:50-51]().
6.  **Answer Generation**: 관련 subgraph를 사용해 답변을 구성하고, 관계, architectural context를 설명하며, concepts를 code locations와 연결합니다. 일치 항목이 없으면 사용자에게 알리고 관련 용어를 제안합니다 [understand-anything-plugin/skills/understand-chat/SKILL.md:53-56]().

## `/understand-diff`

Git diff 또는 pull request를 분석하여 변경 사항, 영향을 받는 구성 요소, 위험을 이해합니다.

### 인자와 플래그

제공된 `SKILL.md`에는 명시적으로 문서화된 내용이 없습니다. 이 스킬은 현재 Git 상태를 암묵적으로 사용합니다.

### 예상 출력

다음을 포함하는 구조화된 변경 분석입니다.
*   **Changed Components**: 직접 수정된 구성 요소와 요약입니다.
*   **Affected Components**: 변경으로 영향을 받는 구성 요소(1-hop dependencies)입니다.
*   **Affected Layers**: 변경이 닿은 architectural layers입니다.
*   **Risk Assessment**: complexity, cross-layer edges, blast radius를 기반으로 합니다.
*   대시보드 시각화를 위한 `.understand-anything/`의 `diff-overlay.json` 파일입니다.

### 구현 세부 사항

`/understand-diff` 스킬은 기존 `knowledge-graph.json`과 Git 정보를 활용합니다.
1.  **Prerequisite Check**: `knowledge-graph.json`이 존재하는지 확인하고, 없으면 사용자에게 `/understand`를 실행하라고 안내합니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:33-34]().
2.  **Changed Files List**: Git 명령을 사용해 변경된 파일을 결정합니다.
    *   commit되지 않은 변경에는 `git diff --name-only`.
    *   feature branches에는 `git diff main...HEAD --name-only`.
    *   지정된 PR 번호에서 diff를 가져옵니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:36-38]().
3.  **Project Metadata**: context를 위해 그래프에서 `project` 섹션을 추출합니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:40-41]().
4.  **Changed Nodes**: 각 변경 파일 경로에 대해 일치하는 `filePath` 값을 가진 nodes(file-level, function, class nodes)를 지식 그래프에서 검색합니다. 이러한 nodes의 `id` 값은 `changedNodeIds`로 수집됩니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:43-47]().
5.  **Affected Edges (1-hop)**: 각 `changedNodeId`에 대해 `edges` 섹션을 검색해 들어오는 연결과 나가는 연결을 찾습니다. 이를 통해 upstream callers와 downstream dependencies가 식별되며, 이들은 "affected components"로 간주됩니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:49-52]().
6.  **Affected Layers**: `changedNodeIds`에 대해 `layers` 섹션을 검색해 어떤 architectural layers가 영향을 받는지 결정합니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:54-55]().
7.  **Structured Analysis**: 변경된 구성 요소, 영향을 받는 구성 요소, 영향을 받는 layers, node `complexity`, cross-layer edges, 영향을 받는 구성 요소 수를 기반으로 한 risk assessment를 다루는 상세 분석 보고서를 생성합니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:57-60]().
8.  **Diff Overlay Output**: `changedFiles`, `changedNodeIds`, `affectedNodeIds`를 포함하는 `diff-overlay.json` 파일을 씁니다. 이 파일은 대시보드에서 diff를 시각적으로 표현하는 데 사용됩니다 [understand-anything-plugin/skills/understand-diff/SKILL.md:62-71](). 이후 사용자는 diff를 시각화하기 위해 `/understand-anything:understand-dashboard`를 사용할 수 있다고 안내받습니다.

## `/understand-explain`

코드베이스의 특정 파일, 함수, 모듈에 대한 심층 설명을 제공합니다.

### 인자와 플래그

*   `[file-path]`: 파일 경로 또는 함수/클래스 이름이 포함된 파일 경로(예: `src/auth/login.ts:verifyToken`)입니다.

### 예상 출력

지정된 코드 구성 요소의 역할, 내부 구조, 외부 연결, 데이터 흐름, 주목할 만한 패턴이나 복잡성을 포함하는 철저하고 심층적인 설명입니다.

### 구현 세부 사항

`/understand-explain` 스킬은 `knowledge-graph.json`과 직접 파일 콘텐츠에 의존합니다.
1.  **Prerequisite Check**: `.understand-anything/knowledge-graph.json`의 존재를 검증하고, 없으면 사용자에게 `/understand`를 실행하라고 안내합니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:34-35]().
2.  **Target Node Identification**: 제공된 인자를 기반으로 `grep`을 사용해 지식 그래프에서 target node를 찾습니다. 여기에는 file paths에 대한 `filePath` 검색 또는 functions/classes에 대한 `name` 필드 검색(file path로 필터링)이 포함됩니다. node의 `id`, `type`, `summary`, `tags`, `complexity`가 기록됩니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:37-40]().
3.  **Connected Edges**: target node의 ID에 대해 `edges` 섹션을 검색해 모든 incoming 및 outgoing connections(예: `calls`, `imports`, `depends_on`)를 찾습니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:42-44]().
4.  **Connected Nodes**: 연결된 각 node ID에 대해 `nodes` 섹션에서 `name`, `summary`, `type`을 가져와 구성 요소의 neighborhood context를 구축합니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:46-47]().
5.  **Layer Context**: `layers` 섹션을 검색해 target node가 속한 architectural layer와 그 설명을 식별합니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:49-50]().
6.  **Source File Content**: 자세한 분석을 위해 node의 `filePath`에 있는 실제 소스 파일 콘텐츠를 읽습니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:52-53]().
7.  **Explanation Generation**: 수집한 모든 정보를 결합해 구성 요소의 역할, 내부 구조, 외부 연결, 데이터 흐름을 설명하고 중요한 패턴이나 복잡성을 강조합니다 [understand-anything-plugin/skills/understand-explain/SKILL.md:55-59]().

## `/understand-domain`

코드베이스에서 비즈니스 도메인 지식을 추출하고 대화형 domain flow graph를 생성합니다.

### 인자와 플래그

*   `--full`: `knowledge-graph.json`이 이미 존재하더라도 fresh scan을 강제합니다.

### 예상 출력

`.understand-anything/` 안의 `domain-graph.json` 파일이며, domains, business flows, process steps를 포함하고 대시보드에서 시각화할 수 있습니다.

### 구현 세부 사항

`/understand-domain` 스킬은 두 가지 모드로 동작할 수 있습니다. 기존 지식 그래프에서 파생하거나 lightweight scan을 수행합니다.
1.  **Phase 0: `PROJECT_ROOT` 해결**: `/understand`와 유사하게 `PROJECT_ROOT`를 해석하고 필요한 경우 worktree redirection을 수행합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:19-40](). 또한 `PLUGIN_ROOT`도 해석합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:46-85]().
2.  **Phase 1: 기존 그래프 감지**: `$PROJECT_ROOT/.understand-anything/knowledge-graph.json`을 확인합니다.
    *   존재하고 `--full`이 전달되지 않았다면 Phase 3(그래프에서 파생)으로 진행합니다.
    *   그렇지 않으면 Phase 2(lightweight scan)로 진행합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:89-93]().
3.  **Phase 2: Lightweight Scan (Path 1)**:
    *   `PROJECT_ROOT`와 함께 `extract-domain-context.py`를 실행합니다. 이 스크립트는 lightweight scan을 수행해 file tree, detected entry points, file signatures, code snippets, project metadata가 포함된 `domain-context.json`을 생성합니다 [understand-anything-anything-plugin/skills/understand-domain/extract-domain-context.py:3-8, 99-108]().
    *   `extract-domain-context.py` 스크립트는 추출 context의 범위와 크기를 제어하기 위해 `MAX_FILE_TREE_DEPTH`, `MAX_FILES_PER_DIR`, `MAX_FILES_TOTAL`, `MAX_SAMPLED_FILES`, `MAX_LINES_PER_FILE`, `MAX_ENTRY_POINTS`, `MAX_OUTPUT_BYTES`를 사용합니다 [understand-anything-plugin/skills/understand-domain/extract-domain-context.py:26-30](). 또한 `.gitignore`를 파싱해 관련 없는 파일을 제외합니다 [understand-anything-plugin/skills/understand-domain/extract-domain-context.py:118-146]().
    *   그런 다음 Phase 4로 진행합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:109-110]().
4.  **Phase 3: Derive from Existing Graph (Path 2)**:
    *   `$PROJECT_ROOT/.understand-anything/knowledge-graph.json`을 읽습니다.
    *   domain analyzer를 위한 구조화된 context로 graph data(nodes, edges, layers, tour steps)를 포맷합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:114-120]().
    *   그런 다음 Phase 4로 진행합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:121-122]().
5.  **Phase 4: Domain Analysis**:
    *   `$PLUGIN_ROOT/agents/domain-analyzer.md`의 prompt와 Phase 2 또는 3의 context를 사용해 `domain-analyzer` subagent를 dispatch합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:125-127]().
    *   에이전트는 출력을 `$PROJECT_ROOT/.understand-anything/intermediate/domain-analysis.json`에 씁니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:128-128]().
6.  **Phase 5: Validate and Save**:
    *   `domain-analysis.json` 출력을 읽습니다.
    *   표준 graph validation pipeline(`domain`, `flow`, `step` node types를 지원)을 사용해 검증합니다.
    *   검증된 그래프를 `$PROJECT_ROOT/.understand-anything/domain-graph.json`에 저장합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:130-134]().
    *   intermediate files를 정리합니다 [understand-anything-plugin/skills/understand-domain/SKILL.md:135-135]().

## `/understand-onboard`

프로젝트에 합류하는 새 팀원을 위한 포괄적인 온보딩 가이드를 생성합니다.

### 인자와 플래그

명시적으로 문서화되어 있지 않습니다.

### 예상 출력

markdown 형식의 온보딩 가이드이며, 잠재적으로 `docs/ONBOARDING.md`에 저장될 수 있습니다.

### 구현 세부 사항

`/understand-onboard` 스킬은 `knowledge-graph.json`에서 정보를 추출하여 온보딩 가이드를 구성합니다.
1.  **Prerequisite Check**: `knowledge-graph.json`이 존재하는지 확인하고, 없으면 사용자에게 `/understand`를 실행하라고 안내합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:32-33]().
2.  **Project Metadata**: 개요를 위해 `project` 섹션(name, description, languages, frameworks)을 추출합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:35-36]().
3.  **Layers**: 가이드의 architectural structure를 정의하기 위해 `layers` 배열을 가져옵니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:38-39]().
4.  **Tour**: guided walkthrough steps를 위해 `tour` 섹션을 추출합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:41-42]().
5.  **File-level Nodes**: 가이드를 상위 수준으로 유지하기 위해 `grep`을 사용해 file-level nodes(`file`, `config`, `document`, `service`, `pipeline`, `table`, `schema`, `resource`, `endpoint`)만 찾습니다. 이러한 nodes에서 `name`, `filePath`, `summary`, `complexity`를 추출합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:44-46]().
6.  **Complexity Hotspots**: 주의 깊게 살펴야 할 영역을 강조하기 위해 file-level nodes에서 높은 `complexity` 값을 가진 nodes를 식별합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:48-49]().
7.  **Guide Generation**: Project Overview, Architecture Layers, Key Concepts, Guided Tour, File Map, Complexity Hotspots 섹션으로 온보딩 가이드를 조립하고 깔끔한 markdown으로 포맷합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:51-52]().
8.  **Saving and Suggestion**: 가이드를 `docs/ONBOARDING.md`에 저장할 것을 제안하고 저장소에 commit할 것을 제안합니다 [understand-anything-plugin/skills/understand-onboard/SKILL.md:53-54]().
