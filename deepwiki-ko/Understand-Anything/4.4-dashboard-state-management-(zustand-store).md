이 페이지는 `@understand-anything/dashboard` 패키지에 위치한 Understand Anything dashboard의 핵심 state management 구현을 문서화합니다. dashboard는 Zustand를 활용하여 UI state, graph data, navigation logic, filtering preference, search functionality를 캡슐화하는 중앙 reactive store를 정의합니다. 이 페이지는 `useDashboardStore` Zustand store의 주요 기능을 다룹니다.

- Graph loading 및 indexing(`setGraph`)
- Navigation, node selection, layered drill-down(`selectNode`, `drillIntoLayer`, `nodeHistory`)
- Persona filtering(user expertise level)
- Graph view의 detail level toggling(file-level vs. class-level)
- Code neighborhood를 isolate하기 위한 focus mode
- Diff overlay state 통합
- `SearchEngine` 및 `SemanticSearchEngine` subsystem을 통한 search integration.

---

## 1. 목적과 범위

`useDashboardStore` Zustand store는 dashboard state의 single source of truth 역할을 합니다. loaded knowledge graph data를 관리하고 효율적인 lookup을 위해 indexing하며, 현재 node selection과 navigation context를 추적하고, UI panel 및 mode(예: focus mode, detail level)를 제어하며, filtering과 search mode를 관리합니다. 또한 내장 search engine과 통합되어 fuzzy 및 semantic search 기능을 모두 제공합니다.

이 페이지는 dashboard가 이 state를 유지하고 update하는 주요 function, state field, mechanism을 설명하며, UI에 노출되는 interactive graph exploration 및 filtering 기능을 지원하는 방식을 다룹니다.

## 2. Store Structure 및 Core Data

### 2.1 Graph Data 및 Indexing

- `graph`는 현재 `KnowledgeGraph` 객체를 보관하며, 로드된 graph가 없으면 null입니다.
- 모든 `setGraph` 호출 시 indexed cache가 다시 빌드됩니다.
  - `nodesById`: ID로 node를 효율적으로 lookup하기 위한 `Map<string, GraphNode>`.
  - `nodeIdToLayerId`: node ID를 단일 primary layer ID(처음 matching되는 layer)에 mapping하는 `Map<string, string>`.
  - `nodeIdToLayerIds`: node ID를 해당 node가 나타나는 모든 layer에 mapping하는 `Map<string, Set<string>>`.

이 index들을 구축하면 filtering과 navigation을 위한 빠른 node retrieval 및 layer membership query가 가능합니다.

```ts
function buildGraphIndexes(graph: KnowledgeGraph): {
  nodesById: Map<string, GraphNode>;
  nodeIdToLayerId: Map<string, string>;
  nodeIdToLayerIds: Map<string, Set<string>>;
} {
  const nodesById = new Map<string, GraphNode>();
  for (const node of graph.nodes) nodesById.set(node.id, node);
  const nodeIdToLayerId = new Map<string, string>();
  const nodeIdToLayerIds = new Map<string, Set<string>>();
  for (const layer of graph.layers) {
    for (const nid of layer.nodeIds) {
      if (!nodeIdToLayerId.has(nid)) nodeIdToLayerId.set(nid, layer.id);
      let set = nodeIdToLayerIds.get(nid);
      if (!set) {
        set = new Set<string>();
        nodeIdToLayerIds.set(nid, set);
      }
      set.add(layer.id);
    }
  }
  return { nodesById, nodeIdToLayerId, nodeIdToLayerIds };
}
```

### 2.2 Core State Fields

- `selectedNodeId`: 현재 선택된 graph node 또는 null입니다.
- `nodeHistory`: 방문한 node ID 배열이며, 최대 50개 entry로 제한됩니다.
- `navigationLevel`: `"overview"` 또는 `"layer-detail"`이며, dashboard lens depth를 제어합니다.
- `activeLayerId`: 현재 drill into된 layer입니다.
- `persona`: `"non-technical"`, `"junior"`, `"experienced"` 중 user filtering preference입니다.
- `detailLevel`: `"file"` 또는 `"class"` display detail level입니다.
- `focusNodeId`: 설정된 경우 node의 1-hop neighborhood를 선택적으로 isolate합니다.
- `diffMode`, `changedNodeIds`, `affectedNodeIds`: diff overlay highlighting을 관리합니다.

### 2.3 Filter State 및 Node Category Toggling

store는 다음을 포함하는 풍부한 filter state를 유지합니다.

- set으로 표현되는 node type, complexity, layer, edge category.
- filtering UI를 단순화하기 위한 node type의 category-based toggle:
  
```ts
nodeTypeFilters: Record<NodeCategory, boolean>;
toggleNodeTypeFilter: (category: NodeCategory) => void;
```

## 3. 주요 Actions 및 Functions

### 3.1 Graph Data Loading 및 Setting: `setGraph`

`setGraph` action은 전체 knowledge graph를 교체하고 모든 내부 index를 다시 빌드합니다.

- cache(`nodesById`, `nodeIdToLayerId`, `nodeIdToLayerIds`)를 update하기 위해 `buildGraphIndexes`를 호출합니다.
- selection, history, active layer, filter, diff, search engine instance를 reset합니다.
- 새 graph로 fuzzy 및 semantic `SearchEngine` instance를 적절히 initialize합니다.

이를 통해 새 graph load 또는 refresh 시 clean reinitialization이 가능합니다.

### 3.2 Navigation, Selection 및 History

- `selectNode(nodeId: string | null)`: 현재 node selection을 설정합니다.
- `navigateToNode(nodeId: string)`: 주어진 node로 navigate합니다. 내부적으로 `navigateToNodeInLayer`를 호출합니다.
- `navigateToNodeInLayer(nodeId: string)`: `nodeIdToLayerId`를 통해 node가 속한 layer를 결정하고, 그에 따라 `activeLayerId`를 설정하며, `navigationLevel`을 `"layer-detail"`로 설정하고 node를 선택합니다.

store는 `nodeHistory`를 이전 방문 node의 stack으로 유지하며, history를 되돌리는 method(`goBackNode`)와 history item으로 직접 navigate하는 method(`navigateToHistoryIndex`)를 제공합니다.

### 3.3 Layer Drill Into 및 Navigation

layer로 drill into하면 graph view가 해당 layer의 detail로 zoom됩니다.

- `drillIntoLayer(layerId: string)`: `activeLayerId`와 navigation level을 `"layer-detail"`로 설정합니다.
- `navigateToOverview()`: navigation level을 `"overview"`로 reset하고 `activeLayerId`를 null로 설정합니다.

### 3.4 Persona Filtering

store는 `persona` state를 추적합니다.

- 세 단계: `"non-technical"`, `"junior"`, `"experienced"`.
- component는 이를 사용해 표시되는 content를 filter하거나 UI explanation을 조정합니다.
- `setPersona` action이 persona state를 update합니다.

### 3.5 Detail Level: File vs. Class

dashboard는 code 표현을 위한 두 가지 detail level을 제공합니다.

- `"file"`: file node만 표시하는 기본 architecture-level view입니다.
- `"class"`: class와 선택적으로 function을 표시하는 code-structure-level view입니다.

State field:

- `detailLevel`: 현재 level입니다.
- `showFunctionsInClassView`: class 내부 function expansion을 위한 boolean toggle입니다.
- `setDetailLevel` 및 `toggleShowFunctionsInClassView` action이 이 설정을 제어합니다.

### 3.6 Focus Mode

- `focusNodeId`는 1-hop neighborhood view를 isolate할 node의 ID를 보관합니다.
- focus mode를 활성화하면 선택된 node의 immediate code context를 isolate하는 데 도움이 됩니다.
- `setFocusNode`는 이 state를 clear하거나 설정합니다.

### 3.7 Diff Overlay State

code change overlay visualization을 지원합니다.

- `diffMode`: overlay on/off toggle입니다.
- `changedNodeIds`와 `affectedNodeIds`: 변경되었거나 transitively affected된 node ID set입니다.
- diff toggling 및 clearing 관련 action을 제공합니다.

### 3.8 Search Integration

- `searchQuery`: 현재 search string입니다.
- `searchResults`: 현재 result(`SearchResult[]` type)입니다.
- `searchMode`: `"fuzzy"` 또는 `"semantic"`입니다.
- `searchEngine`: `SearchEngine` 또는 `SemanticSearchEngine`의 instance입니다.
- `setSearchMode`는 search mode를 전환하고 engine을 reinitialize합니다.
- search update는 query와 mode에 따라 `searchResults`를 update합니다.

store는 semantic search backend를 연결하고 UI에서 fuzzy와 semantic result 사이의 fallback을 관리합니다.

## 5. 상세 Function 설명

### 5.1 `setGraph(graph: KnowledgeGraph)`

- dashboard 전반에서 사용되는 전체 graph state를 reset합니다.
- 효율적인 lookup을 생성하기 위해 `buildGraphIndexes(graph)`를 호출합니다.
- 새 graph data로 search engine을 initialize합니다.
- filter, selection, navigation, diff state를 reset합니다.
- UI가 새 graph data에 즉시 반응할 수 있게 합니다.

### 5.2 `drillIntoLayer(layerId: string)`

- lens focus를 특정 code layer로 변경합니다.
- `activeLayerId`를 target으로 설정하고 `navigationLevel`을 `"layer-detail"`로 설정합니다.
- 사용자가 overview에서 detailed layer content로 drill down할 수 있게 합니다.

### 5.3 `selectNode(nodeId: string | null)`

- 현재 user selection을 반영하도록 `selectedNodeId`를 update합니다.
- 또한 `nodeHistory`에 append하며, `MAX_HISTORY = 50` entry로 제한됩니다.
- null을 선택하면 focus를 clear합니다.

### 5.4 `nodeHistory` 및 Backward Navigation

- `nodeHistory`는 방문한 node(ID)의 history stack을 기록합니다.
- `goBackNode()`는 가장 최근 node를 pop하고 이전 node를 다시 선택합니다.
- `navigateToHistoryIndex(index: number)`는 특정 entry로 jump합니다.

### 5.5 `setPersona(persona: Persona)`

- user expertise에 따라 표시되는 node 또는 content를 filter합니다.
- node rendering의 visibility 또는 complexity를 toggle하는 데 사용됩니다.

### 5.6 `setDetailLevel(level: DetailLevel)` 및 `toggleShowFunctionsInClassView()`

- "file" overview와 더 상세한 "class" view 사이를 toggle할 수 있게 합니다.
- detail level이 class일 때 class 아래 function을 선택적으로 표시합니다.

### 5.7 Focus Mode State Management

- `focusNodeId`는 graph visualization을 단일 node 및 immediate neighbor로 isolate합니다.
- 특정 code region의 심층 탐색에 사용됩니다.

### 5.8 Search Integration 및 Wiring

- 초기에는 `searchEngine`을 `SearchEngine` instance(fuzzy용)로 설정합니다.
- embedding에 대한 cosine similarity를 위해 `SemanticSearchEngine`으로 전환할 수 있습니다.
- `searchQuery` update는 search를 시작하고 `searchResults`를 채웁니다.
- result update는 UI highlight 및 selection logic을 trigger합니다.

## 7. 요약

dashboard의 Zustand store(`useDashboardStore`)는 포괄적이고 성능 좋은 state management backbone을 제공합니다.

- 빠른 node lookup과 layer determination을 위한 효율적인 graph indexing 및 caching.
- drill-down과 focus를 지원하는 풍부한 navigation 및 history management.
- complexity와 explanation을 조정하기 위한 persona 기반 user filtering.
- coarse 또는 fine code view를 위한 detail level toggling.
- code change를 명확히 시각화하기 위한 diff overlay state.
- fuzzy와 semantic embedding 모두에 연결된 dual-mode search integration.
- code viewer와 panel toggle을 위한 centralized UI state.

이 설계는 dashboard component(graph view, search bar, filter, file explorer, persona selector 등)가 user action과 backend data를 reactive하고 synchronous하게 반영하도록 하며, 큰 knowledge graph 탐색을 매끄럽게 안내합니다.
