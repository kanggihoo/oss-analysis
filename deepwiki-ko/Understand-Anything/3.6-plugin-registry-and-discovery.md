## AnalyzerPlugin 인터페이스

plugin 시스템의 핵심은 [understand-anything-plugin/packages/core/src/types.ts]()에 정의된 `AnalyzerPlugin` 인터페이스입니다. 이 인터페이스는 구조 분석 기능을 제공하기 위해 모든 plugin이 구현해야 하는 메서드를 지정합니다.

```typescript
// understand-anything-plugin/packages/core/src/types.ts
export interface AnalyzerPlugin {
  readonly name: string;
  readonly languages: string[]; // List of language IDs this plugin supports

  /**
   * Performs structural analysis on a file.
   * @param filePath The path to the file.
   * @param content The content of the file.
   * @returns StructuralAnalysis object or null if not applicable.
   */
  analyzeFile(filePath: string, content: string): StructuralAnalysis | null;

  /**
   * Resolves import statements within a file.
   * @param filePath The path to the file.
   * @param content The content of the file.
   * @returns Array of ImportResolution objects or null if not applicable.
   */
  resolveImports?(filePath: string, content: string): ImportResolution[] | null;

  /**
   * Extracts call graph entries from a file.
   * @param filePath The path to the file.
   * @param content The content of the file.
   * @returns Array of CallGraphEntry objects or null if not applicable.
   */
  extractCallGraph?(filePath: string, content: string): CallGraphEntry[] | null;
}
```

`AnalyzerPlugin` 인터페이스는 `name`과 해당 plugin이 지원하는 `languages` 목록을 요구합니다. 세 가지 주요 메서드를 정의합니다.
*   `analyzeFile`: 함수, 클래스, imports, exports를 추출하는 깊은 구조 분석을 수행합니다.
*   `resolveImports`: import 문 해석을 전담합니다. 일부 plugin(예: 비코드 파일용 plugin)은 import를 해석할 필요가 없을 수 있으므로 이 메서드는 선택 사항입니다.
*   `extractCallGraph`: call graph 정보를 추출하며, 이 역시 선택 메서드입니다.

## PluginRegistry

`PluginRegistry` 클래스([understand-anything-plugin/packages/core/src/plugins/registry.ts:11-81]())는 `AnalyzerPlugin` 인스턴스를 관리하고 dispatch하는 중앙 허브 역할을 합니다. 등록된 plugin 모음을 유지하고, 지원 언어를 해당 plugin에 매핑합니다.

### 생성자와 초기화

`PluginRegistry`는 선택적인 `LanguageRegistry` 인스턴스로 초기화할 수 있습니다. 제공되지 않으면 기본값으로 `LanguageRegistry.createDefault()`를 사용합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:18](). 이 `LanguageRegistry`는 파일 확장자를 language ID에 매핑하는 데 중요하며, 그 결과 `PluginRegistry`가 주어진 파일을 어떤 plugin이 처리해야 하는지 결정하는 데 도움을 줍니다.

```typescript
// understand-anything-plugin/packages/core/src/plugins/registry.ts
export class PluginRegistry {
  private plugins: AnalyzerPlugin[] = [];
  private languageMap = new Map<string, AnalyzerPlugin>();
  private languageRegistry: LanguageRegistry;

  constructor(languageRegistry?: LanguageRegistry) {
    this.languageRegistry = languageRegistry ?? LanguageRegistry.createDefault();
  }
  // ...
}
```

### Plugin 관리

*   **`register(plugin: AnalyzerPlugin): void`**: registry에 plugin을 추가합니다. plugin이 지원하는 각 언어에 대해 내부 `languageMap`에 항목이 추가됩니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:20-24](). 여러 plugin이 같은 언어를 지원하면, 해당 언어에 대해 마지막에 등록된 plugin이 우선합니다 [understand-anything-plugin/packages/core/src/__tests__/plugin-registry.test.ts:85-92]().
*   **`unregister(name: string): void`**: `name`으로 plugin을 제거합니다. 이 메서드는 `languageMap`을 다시 빌드하여 제거된 plugin이 이전에 처리하던 언어가 올바르게 재할당되거나 제거되도록 보장합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:26-37]().
*   **`getPlugins(): AnalyzerPlugin[]`**: 현재 등록된 모든 plugin 목록을 반환합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:74-76]().
*   **`getSupportedLanguages(): string[]`**: 등록된 plugin들이 지원하는 모든 고유 language ID 목록을 반환합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:78-80]().

### Plugin Discovery와 호출

`PluginRegistry`는 언어 또는 파일 경로를 기준으로 plugin을 찾고 호출하는 메서드를 제공합니다.

*   **`getPluginForLanguage(language: string): AnalyzerPlugin | null`**: 특정 language ID에 등록된 plugin을 가져옵니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:39-41]().
*   **`getPluginForFile(filePath: string): AnalyzerPlugin | null`**: `LanguageRegistry`를 사용해 주어진 `filePath`의 언어를 결정한 다음 해당 plugin을 반환합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:43-47]().
*   **`getLanguageForFile(filePath: string): string | null`**: 주어진 파일 경로에 대한 language ID를 가져오는 helper 메서드입니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:51-53]().
*   **`analyzeFile(filePath: string, content: string): StructuralAnalysis | null`**: `filePath`에 적합한 plugin을 찾고 `analyzeFile` 호출을 그 plugin에 위임합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:55-59]().
*   **`resolveImports(filePath: string, content: string): ImportResolution[] | null`**: `analyzeFile`과 유사하지만 발견된 plugin의 `resolveImports` 메서드로 위임합니다. plugin이 `resolveImports`를 구현하지 않았을 수 있는 경우도 처리합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:61-65]().
*   **`extractCallGraph(filePath: string, content: string): CallGraphEntry[] | null`**: 발견된 plugin의 `extractCallGraph` 메서드로 위임하며, 선택적 구현도 처리합니다 [understand-anything-plugin/packages/core/src/plugins/registry.ts:67-71]().

## Plugin Discovery와 설정

시스템은 JSON 파일을 통한 동적 plugin 설정을 지원하여 사용자가 plugin을 활성화/비활성화하고, 지원 언어를 지정하며, custom option을 전달할 수 있게 합니다.

### PluginConfig 인터페이스

plugin 설정 구조는 `PluginConfig`와 `PluginEntry` 인터페이스로 정의됩니다 [understand-anything-plugin/packages/core/src/plugins/discovery.ts:3-12]().

```typescript
// understand-anything-plugin/packages/core/src/plugins/discovery.ts
export interface PluginEntry {
  name: string;
  enabled: boolean;
  languages: string[];
  options?: Record<string, unknown>;
}

export interface PluginConfig {
  plugins: PluginEntry[];
}
```

*   `PluginEntry`: 단일 plugin의 설정을 나타내며, `name`, `enabled` 상태, 처리해야 할 `languages` 목록, 선택적 `options`를 포함합니다.
*   `PluginConfig`: 최상위 설정 객체로, `PluginEntry` 객체 배열을 포함합니다.

### DEFAULT_PLUGIN_CONFIG

기본 설정이 제공되며, `builtinLanguageConfigs` 안에 `treeSitter` 설정이 정의된 모든 언어에 대해 `tree-sitter` plugin을 활성화합니다 [understand-anything-plugin/packages/core/src/plugins/discovery.ts:14-24]().

```typescript
// understand-anything-plugin/packages/core/src/plugins/discovery.ts
export const DEFAULT_PLUGIN_CONFIG: PluginConfig = {
  plugins: [
    {
      name: "tree-sitter",
      enabled: true,
      languages: builtinLanguageConfigs
        .filter((c) => c.treeSitter)
        .map((c) => c.id),
    },
  ],
};
```
이를 통해 시스템은 별도 설정 없이도 일반적인 programming language의 구조 분석에 강력한 `TreeSitterPlugin`을 활용할 수 있습니다.

### `parsePluginConfig`

`parsePluginConfig` 함수([understand-anything-plugin/packages/core/src/plugins/discovery.ts:30-61]())는 JSON 문자열을 `PluginConfig` 객체로 파싱하는 역할을 합니다.

*   비어 있거나 유효하지 않은 JSON 문자열은 `DEFAULT_PLUGIN_CONFIG`를 반환하여 처리합니다 [understand-anything-plugin/packages/core/src/plugins/discovery.ts:31-32, 58-60]().
*   파싱된 JSON의 구조를 검증하며, 특히 `plugins`가 배열인지와 각 `PluginEntry`에 필수 `name` 및 `languages` 필드가 있는지 확인합니다 [understand-anything-plugin/packages/core/src/plugins/discovery.ts:34-48](). 이러한 필드가 없는 항목은 필터링됩니다 [understand-anything-plugin/packages/core/src/__tests__/plugin-discovery.test.ts:36-43]().
*   JSON에서 `enabled` 필드가 생략되면 기본값은 `true`입니다 [understand-anything-plugin/packages/core/src/plugins/discovery.ts:52]().

### `serializePluginConfig`

`serializePluginConfig` 함수([understand-anything-plugin/packages/core/src/plugins/discovery.ts:66-68]())는 `PluginConfig` 객체를 사람이 읽기 쉬운 JSON 문자열로 다시 변환하며, 2-space indentation으로 형식화합니다.

```typescript
// understand-anything-plugin/packages/core/src/plugins/discovery.ts
export function serializePluginConfig(config: PluginConfig): string {
  return JSON.stringify(config, null, 2);
}
```

## 예시: TreeSitterPlugin

`TreeSitterPlugin`([understand-anything-plugin/packages/core/src/plugins/tree-sitter-plugin.ts:31-97]())은 `AnalyzerPlugin` 구현의 대표적인 예입니다. `web-tree-sitter`를 활용하여 다양한 programming language에 대해 매우 정확한 구조 분석을 제공합니다.

*   **생성자**: 지원할 언어와 WASM grammar를 로드하는 방법을 결정하기 위해 `LanguageConfig` 객체를 받습니다. 또한 지원 언어마다 `LanguageExtractor` 인스턴스를 등록합니다 [understand-anything-plugin/packages/core/src/plugins/tree-sitter-plugin.ts:56-97]().
*   **`init()`**: 이 비동기 메서드는 `web-tree-sitter` WASM 모듈과 필요한 모든 언어 grammar를 로드합니다. 분석을 수행하기 전에 반드시 호출되어야 합니다 [understand-anything-plugin/packages/core/src/plugins/tree-sitter-plugin.ts:124-197]().
*   **`analyzeFile()`**: 이 메서드는 로드된 Tree-sitter grammar와 언어별 `LanguageExtractor`를 사용해 파일 내용을 파싱하고 함수, 클래스, imports, exports 같은 구조 정보를 추출합니다 [understand-anything-plugin/packages/core/src/plugins/tree-sitter-plugin.ts:203-220]().
*   **`resolveImports()`**: import path 해석을 언어별 extractor에 위임합니다 [understand-anything-plugin/packages/core/src/plugins/tree-sitter-plugin.ts:222-226]().
*   **`extractCallGraph()`**: call graph 구축을 언어별 extractor에 위임합니다 [understand-anything-plugin/packages/core/src/plugins/tree-sitter-plugin.ts:228-232]().

`TreeSitterPlugin`은 정적 분석을 수행하는 강력한 language-agnostic 방식이며, 언어별 세부 사항은 `LanguageExtractor` 구현체가 처리합니다.
