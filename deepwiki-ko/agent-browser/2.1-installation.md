## 개요

`agent-browser` package는 두 가지 주요 구성 요소로 이루어져 있습니다.

1.  **Native CLI Binary**: daemon lifecycle을 관리하고, 명령을 parsing하며, CDP를 통해 통신하는 100% Rust executable입니다 [docs/src/app/installation/page.mdx:5-12]().
2.  **Chromium Browser**: CLI의 내부 installer가 관리하는 "Chrome for Testing" build입니다 [cli/src/install.rs:7-15]().

CLI binary는 npm 설치 중 `postinstall.js` script를 통해 자동으로 다운로드됩니다 [scripts/postinstall.js:4-10](). Chromium browser는 `agent-browser install` 명령으로 설치됩니다 [scripts/postinstall.js:205]().

### 지원 플랫폼
Native Rust binary는 다양한 architecture용으로 제공되며 GitHub Actions workflow를 사용해 build됩니다 [ .github/workflows/release.yml:71-117]().

| 플랫폼 | Architecture | Binary Name |
| :--- | :--- | :--- |
| Linux (glibc) | x64 | `agent-browser-linux-x64` |
| Linux (glibc) | ARM64 | `agent-browser-linux-arm64` |
| Linux (musl/Alpine) | x64 | `agent-browser-linux-musl-x64` |
| Linux (musl/Alpine) | ARM64 | `agent-browser-linux-musl-arm64` |
| macOS | x64 | `agent-browser-darwin-x64` |
| macOS | ARM64 | `agent-browser-darwin-arm64` |

## 설치 방법

### 전역 설치(권장)
전역 설치는 npm shim을 patch하여 native Rust binary를 직접 호출할 수 있게 하므로, 오버헤드 없는 실행이 가능합니다 [scripts/postinstall.js:218-220]().

```bash
npm install -g agent-browser
agent-browser install  # Download Chrome
```

### 로컬 프로젝트 설치
`package.json`에서 특정 version을 고정하는 프로젝트의 경우:

```bash
npm install agent-browser
npx agent-browser install
```

### 소스에서 설치
Node.js 24+, pnpm, Rust toolchain([rustup.rs](https://rustup.rs))이 필요합니다 [docs/src/app/installation/page.mdx:46-58]().

```bash
git clone https://github.com/vercel-labs/agent-browser
cd agent-browser
pnpm install
pnpm build
pnpm build:native  # Compiles cli/Cargo.toml
./bin/agent-browser install
```

출처: [scripts/postinstall.js:146-148](), [bin/agent-browser.js:82-83](), [docs/src/app/installation/page.mdx:48-58](), [.node-version:1]()

## postinstall.js를 통한 Binary 배포

`postinstall.js` script는 플랫폼 감지와 binary retrieval을 처리하여, 사용자가 Rust toolchain 없이도 native executable을 받을 수 있게 합니다.

### 구현 로직
script는 다음 단계를 수행합니다.
1.  **Environment Detection**: `os.platform()`과 `os.arch()`를 사용합니다. 특히 Linux에서 `musl` libc를 확인해 `isMusl()`을 통해 Alpine을 지원합니다 [scripts/postinstall.js:24-36]().
2.  **Download**: `package.json`의 version을 사용해 GitHub Releases에서 binary를 가져옵니다 [scripts/postinstall.js:42-50]().
3.  **Permission Management**: Unix 계열 system에서 binary가 실행 가능하도록 `chmodSync(binaryPath, 0o755)`를 적용합니다 [scripts/postinstall.js:138-140]().
4.  **Install Method Tracking**: `writeInstallMethod()`를 통해 `.install-method` file을 `bin/` directory에 씁니다. 이는 `upgrade` 명령이 update에 `npm`, `pnpm`, `yarn`, `bun` 중 무엇을 사용할지 알기 위해 사용됩니다 [scripts/postinstall.js:91-106]().

### 전역 최적화
전역 설치에서는 script가 `fixGlobalInstallBin()`을 호출해 npm의 bin entry를 patch합니다. 이렇게 하면 Node.js wrapper(`bin/agent-browser.js`)를 우회하고 native binary를 직접 가리킵니다 [scripts/postinstall.js:7-10]().
*   **Windows**: `.cmd` 및 `.ps1` shim을 덮어씁니다 [scripts/postinstall.js:8-9]().
*   **Mac/Linux**: symlink를 native binary를 가리키도록 교체합니다 [scripts/postinstall.js:9-10]().

## Chromium 설정

`agent-browser install` 명령은 호환되는 browser 획득을 관리합니다.

### 구현: install.rs
CLI는 Chrome을 bundle하지 않습니다. 대신 "Chrome for Testing"을 local directory인 `~/.agent-browser/browsers/`에 다운로드합니다 [cli/src/install.rs:10-15]().

1.  **Version Lookup**: `LAST_KNOWN_GOOD_URL`에서 최신 stable version을 가져옵니다 [cli/src/install.rs:7-8]().
2.  **Discovery**: `find_installed_chrome` function은 플랫폼의 naming convention과 일치하는 binary를 찾기 위해 directory를 scan합니다 [cli/src/install.rs:17-91]().
3.  **Platform Specifics**: `chrome_binary_in_dir`는 `Google Chrome for Testing.app`(macOS), `chrome`(Linux), `chrome.exe`(Windows)의 path를 처리합니다 [cli/src/install.rs:101-152]().

### Linux System Dependencies
Linux에서 Chromium은 특정 system library가 필요합니다. `install` 명령의 `--with-deps` flag는 dependency 자동 설치를 trigger합니다 [scripts/postinstall.js:210](), [docs/src/app/installation/page.mdx:60-66]().

## 업데이트와 문제 해결

### Upgrade Command
`agent-browser upgrade` 명령은 설치 방법(예: npm, pnpm, cargo, brew)을 자동으로 감지하고 올바른 update command를 실행합니다 [cli/src/upgrade.rs:142-185](). 진행 전에 npm registry를 확인하여 더 새로운 version이 있는지 검사합니다 [cli/src/upgrade.rs:18-32]().

### Doctor Command
`agent-browser doctor` 명령은 설치 문제를 진단하고 오래된 daemon file을 자동으로 정리합니다 [docs/src/app/installation/page.mdx:78-82]().

다음 항목을 확인합니다.
*   **Environment**: CLI version, platform, disk space [docs/src/app/installation/page.mdx:96]().
*   **Chrome**: install path, version, profile count [docs/src/app/installation/page.mdx:97]().
*   **Daemons**: 실행 중인 session과 오래된 `.sock` 또는 `.pid` file [docs/src/app/installation/page.mdx:98]().
*   **Security**: encryption key와 state file permission [docs/src/app/installation/page.mdx:100]().
*   **Network**: Chrome for Testing CDN 및 AI Gateway의 reachability [docs/src/app/installation/page.mdx:102]().

`--fix` flag를 사용하면 Chrome이 없을 때 도구가 `agent-browser install`을 자동으로 실행하거나 `~/.agent-browser/.encryption-key`에 새 encryption key를 생성할 수 있습니다 [docs/src/app/installation/page.mdx:111-118]().
