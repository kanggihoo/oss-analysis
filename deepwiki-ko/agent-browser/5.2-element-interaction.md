이 페이지는 page element와 상호작용하는 command를 문서화합니다. clicking, typing, form filling, scrolling, hovering, dragging, selecting을 다룹니다. 이 command들은 snapshot의 element reference(ref) 또는 CSS selector를 받습니다. 시스템은 native CDP 기반 실행을 위해 `interaction.rs`의 specialized dispatch pipeline을 사용합니다.

navigation command(open, back, forward, reload)는 [Navigation and Browser Control](5.1)을 참조하세요. element property와 content를 가져오는 방법은 [Information Retrieval](5.3)을 참조하세요.

## Element Targeting

모든 interaction command에는 element selector가 필요하며, 다음 중 하나를 사용할 수 있습니다.

1.  **Element references (refs)**: `snapshot` command 중 생성되는 `@e1`, `@e2` 등.
2.  **CSS selectors**: `#id`, `.class`, `button.primary`.

**ref 사용을 강력히 권장합니다.** ref는 snapshot 시점에 validate되며 DOM attribute가 변경되어도 AI 에이전트에 안정적인 targeting을 제공하기 때문입니다. CLI는 `parse_ref`에서 이러한 ref를 parse하며, `@e1`, `ref=e1`, 또는 단순히 `e1` 같은 format을 지원합니다. [cli/src/native/element.rs:124-147]()

### Ref Resolution Pipeline (Native)

native Rust daemon에서 ref는 마지막 snapshot 중 발견된 element에 대한 metadata를 저장하는 `RefMap`을 사용해 resolve됩니다. [cli/src/native/element.rs:18-21]()

**Title: Ref Resolution and Interaction Pipeline**
```mermaid

    "execute_command" -- "DaemonState" --> "interaction_click"["interaction::click()"]
    "interaction_click" --> "resolve_element_center"["element::resolve_element_center()"]

```
**출처:** [cli/src/native/interaction.rs:28-58](), [cli/src/native/element.rs:149-202](), [cli/src/native/snapshot.rs:216-223]()

`resolve_element_center` function은 cached `backend_node_id`를 사용하는 "fast path"를 구현합니다. node가 stale한 경우(예: page re-render), 저장된 role, name, index(`nth`) 기반 search로 fallback하여 element를 다시 찾습니다. [cli/src/native/element.rs:9-16]()

### Ref Lifecycle

ref는 DOM이 크게 변경되면 **invalidated**됩니다.
*   navigation 이후(link click, form submission).
*   dynamic content load 이후(modal, dropdown).
*   page structure를 수정하는 모든 action 이후.

**page 변경 후에는 항상 다시 snapshot을 생성하세요.**
```bash
agent-browser snapshot -i          # Get refs: @e1, @e2
agent-browser click @e2            # Navigate to new page
agent-browser snapshot -i          # MUST re-snapshot to get fresh @e refs
```
**출처:** [skill-data/core/templates/form-automation.sh:24-54]()

## Mouse Interactions

### Click & Double Click
```bash
agent-browser click <selector> [--new-tab]
agent-browser dblclick <selector>
```
native implementation은 element의 center point를 resolve하고 CDP를 통해 `Input.dispatchMouseEvent`를 dispatch합니다. [cli/src/native/interaction.rs:28-58]() double click은 `click_count: 2`가 설정된 `click`으로 구현됩니다. [cli/src/native/interaction.rs:60-77]()

### Hover
```bash
agent-browser hover <selector>
```
`Input.dispatchMouseEvent`를 통해 `mouseMoved` event type을 사용하여 mouse를 element center로 이동합니다. [cli/src/native/interaction.rs:79-112]()

## Keyboard Interactions

### Fill vs Type
시스템은 text 입력을 위한 두 가지 구분된 방식을 제공합니다.

| Command | Implementation | Behavior |
| :--- | :--- | :--- |
| `fill` | `Runtime.callFunctionOn` + `Input.insertText` | JS로 input을 clear하고 focus한 뒤 text를 insert합니다. 큰 input에 더 빠르고 안정적입니다. [cli/src/native/interaction.rs:114-178]() |
| `type` | `Input.dispatchKeyEvent` (per char) | 실제 key press를 simulate합니다. 모든 character에 대해 `keydown`/`keyup` event를 trigger합니다. [cli/src/native/interaction.rs:181-237]() |

**출처:** [cli/src/native/interaction.rs:114-237]()

### Press
```bash
agent-browser press <key> [selector]
```
단일 key press(예: `Enter`, `Control+a`)를 dispatch합니다. selector가 제공되면 먼저 해당 element에 focus합니다. [cli/src/native/interaction.rs:239-270]()

## Form & Content Control

### Select
```bash
agent-browser select <selector> <value...>
```
`<select>` element에 사용됩니다. multi-select input을 위해 여러 value를 지원합니다. [skill-data/core/templates/form-automation.sh:35]()

### Check / Uncheck
```bash
agent-browser check <selector>
agent-browser uncheck <selector>
```
checkbox와 radio button을 위한 command입니다. [skill-data/core/templates/form-automation.sh:36-37]()

### Upload
```bash
agent-browser upload <selector> <file_paths...>
```
하나 이상의 file을 upload하기 위해 `<input type="file">` element를 target합니다. [skill-data/core/templates/form-automation.sh:39]()

## Scrolling

### Scroll Command
```bash
agent-browser scroll <direction> [amount] [--selector <sel>]
```
**Directions:** `up`, `down`, `left`, `right`.
selector가 제공되면 시스템은 해당 element를 scroll합니다. 그렇지 않으면 window를 scroll합니다.

### Scroll Into View
```bash
agent-browser scrollintoview <selector>
```
element가 보일 때까지 browser가 scroll하도록 강제합니다. 이는 screenshot을 찍거나 현재 viewport에 없는 element를 click하기 전에 중요합니다. [cli/src/native/element.rs:222-223]()

## 구현: interaction.rs Pipeline

native Rust daemon은 `CdpClient`를 통해 Chrome DevTools Protocol(CDP)과 직접 통신하여 interaction을 처리합니다. [cli/src/native/interaction.rs:5-8]()

**Title: Data Flow for Fill Command**
```mermaid
graph LR
    "Fill_Request"["Fill Command Request"] --> "resolve_id"["element::resolve_element_object_id()"]
    "resolve_id" --> "Focus_JS"["CDP: Runtime.callFunctionOn (focus)"]
    "Focus_JS" --> "Clear_JS"["CDP: Runtime.callFunctionOn (clear/input event)"]
    "Clear_JS" --> "Insert_CDP"["CDP: Input.insertText"]
    "Insert_CDP" --> "Interaction_Complete"["Success Response"]
```
**출처:** [cli/src/native/interaction.rs:114-178]()

### `interaction::fill`의 Logic Flow
1.  **Resolve ID**: CDP `RemoteObjectId`를 얻기 위해 `resolve_element_object_id`를 호출합니다. [cli/src/native/interaction.rs:122-129]()
2.  **Focus**: `Runtime.callFunctionOn`을 통해 `this.focus()`를 실행합니다. [cli/src/native/interaction.rs:132-144]()
3.  **Clear**: 모든 text를 select하고, `this.value = ''`를 설정하며, `input` event를 dispatch하는 JS snippet을 실행합니다. [cli/src/native/interaction.rs:147-164]()
4.  **Insert**: 새 value를 넣기 위해 `Input.insertText`를 호출합니다. [cli/src/native/interaction.rs:167-175]()

### `interaction::click`의 Logic Flow
1.  **Resolve Coordinates**: `(x, y)` coordinate와 effective `session_id`를 얻기 위해 `resolve_element_center`를 호출합니다. [cli/src/native/interaction.rs:37-44]()
2.  **Dispatch**: `dispatch_click`을 사용해 해당 coordinate에서 `mousePressed`와 `mouseReleased` event를 browser에 보냅니다. [cli/src/native/interaction.rs:48-57]()

### Cross-Origin Iframes
interaction command는 iframe을 자동으로 처리합니다. ref가 iframe 내부 element에 속하는 경우 `RefEntry`는 `frame_id`를 저장합니다. [cli/src/native/element.rs:15-16]() interaction logic은 command를 dispatch하기 전에 resolution helper 내부에서 호출되는 `resolve_ax_session`을 사용해 해당 iframe 전용 CDP `session_id`를 resolve합니다. [cli/src/native/element.rs:9](), [cli/src/native/snapshot.rs:222-223]()
