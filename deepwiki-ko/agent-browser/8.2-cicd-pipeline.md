## 목적과 범위

이 문서는 `agent-browser`의 continuous integration 및 continuous deployment(CI/CD) infrastructure를 설명합니다. GitHub Actions workflow(`ci.yml`, `release.yml`), version synchronization mechanism(`sync-version.js`, `check-version-sync.js`), platform 전반의 testing strategy, 그리고 automated version detection과 cross-platform binary distribution으로 관리되는 release process를 다룹니다.

---

## 개요

CI/CD pipeline은 GitHub Actions를 사용해 일곱 가지 platform target(Linux x64/ARM64, Linux musl x64/ARM64, macOS x64/ARM64, Windows x64)에 걸쳐 `agent-browser`의 testing, build, release를 자동화합니다. pipeline은 `package.json`, `cli/Cargo.toml`, dashboard 사이의 version consistency를 강제하고, 여러 OS environment에서 test를 실행하며, cross-platform binary distribution을 validate합니다.

### Pipeline Trigger

주요 CI workflow [`.github/workflows/ci.yml:1-9`]()는 다음 조건에서 trigger됩니다.
- `main` branch로 push.
- `main`을 target으로 하는 pull request.
- Manual workflow dispatch.

Release workflow [`.github/workflows/release.yml:1-8`]()는 다음 조건에서 trigger됩니다.
- `main` branch로 push.
- Manual workflow dispatch.

Sources: [`.github/workflows/ci.yml:1-9`](), [`.github/workflows/release.yml:1-8`]()

Version synchronization은 monorepo의 모든 component가 같은 version number를 반영하도록 보장합니다. system은 root `package.json`을 single source of truth로 취급합니다 [`scripts/sync-version.js:17-21`](). 변경 사항은 automated script를 통해 Rust CLI와 Next.js dashboard로 propagate됩니다.

### sync-version.js

synchronization script [`scripts/sync-version.js`]()는 다음 작업을 수행합니다.

| Step | Action | Implementation |
|------|--------|---------|
| 1 | `package.json`에서 version 읽기 | `JSON.parse(readFileSync(...))` [`scripts/sync-version.js:18-21`]() |
| 2 | `cli/Cargo.toml` update | Regex replace: `/^version\s*=\s*"[^"]*"/m` [`scripts/sync-version.js:25-45`]() |
| 3 | `packages/dashboard/package.json` update | JSON update [`scripts/sync-version.js:47-57`]() |
| 4 | `cli/Cargo.lock` update | `cargo update -p agent-browser` [`scripts/sync-version.js:59-79`]() |

script는 network request를 피하기 위해 먼저 `--offline`으로 `cargo update`를 시도하고, package가 cache되어 있지 않으면 online mode로 fallback합니다 [`scripts/sync-version.js:62-75`]().

### Pre-commit Hook

Husky pre-commit hook [`.husky/pre-commit:1-3`]()은 commit 전마다 version sync를 자동으로 실행합니다.

```bash
node scripts/sync-version.js
git add cli/Cargo.toml cli/Cargo.lock
```

### CI Version Check

`version-sync` job [`.github/workflows/ci.yml:11-24`]()은 `node scripts/check-version-sync.js`를 실행해 consistency를 validate합니다. version이 다르면 CI가 실패하여 mismatched version이 `main` branch에 도달하지 못하게 합니다 [`scripts/check-version-sync.js:42-49`]().

### Rust Testing

#### Primary Rust Testing
`rust` job [`.github/workflows/ci.yml:26-51`]()은 `ubuntu-latest`에서 실행됩니다.
- `rustfmt`와 `clippy`가 포함된 `dtolnay/rust-toolchain@stable`을 사용합니다 [`.github/workflows/ci.yml:33-36`]().
- `cargo fmt` check를 실행합니다 [`.github/workflows/ci.yml:43-44`]().
- common mistake를 잡고 style을 강제하기 위해 `cargo clippy`를 실행합니다 [`.github/workflows/ci.yml:46-47`]().
- `cargo test --profile ci`를 실행합니다 [`.github/workflows/ci.yml:49-50`]().

#### Native E2E Tests
`native-e2e` job [`.github/workflows/ci.yml:106-132`]()은 실제 browser interaction을 validate합니다.
- CLI를 통해 Chrome을 설치합니다: `cargo run -- install --with-deps` [`.github/workflows/ci.yml:123-125`]().
- recording capability를 위해 `ffmpeg`를 설치합니다 [`.github/workflows/ci.yml:127-128`]().
- E2E test를 실행합니다: `cargo test --profile ci e2e -- --ignored --test-threads=1` [`.github/workflows/ci.yml:130-131`]().

### Windows Integration Test

`windows-integration` job [`.github/workflows/ci.yml:133-189`]()은 전체 Windows lifecycle을 validate합니다.
1. **Build**: `x86_64-pc-windows-msvc`용 CLI를 compile합니다 [`.github/workflows/ci.yml:153-154`]().
2. **Install**: network instability를 처리하기 위해 3-attempt retry loop로 `install` command를 test합니다 [`.github/workflows/ci.yml:160-171`]().
3. **Lifecycle**: native binary를 통해 `open`, `snapshot`, `close` command를 실행합니다 [`.github/workflows/ci.yml:173-189`]().

## Release Process

release는 `package.json`의 version이 npm의 현재 version과 다를 때 trigger됩니다 [`release.yml:34-46`]().

### Build Pipeline

`build-binaries` job [`.github/workflows/release.yml:66-191`]()은 CLI를 7개 target으로 cross-compile합니다.
- **Linux**: `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`(`cargo-zigbuild`를 사용하며 glibc 2.28 floor check 포함) [`.github/workflows/release.yml:75-86`](), [`.github/workflows/release.yml:192-201`]().
- **Linux Musl**: `x86_64-unknown-linux-musl`, `aarch64-unknown-linux-musl`(`cargo-zigbuild` 사용) [`.github/workflows/release.yml:87-98`]().
- **Windows**: `x86_64-pc-windows-gnu` [`.github/workflows/release.yml:99-104`]().
- **macOS**: `x86_64-apple-darwin`, `aarch64-apple-darwin` [`.github/workflows/release.yml:105-117`]().

### Publishing

`publish` job [`.github/workflows/release.yml:236-267`]()은 다음을 수행합니다.
1. **Artifact Collection**: 모든 platform binary를 download합니다 [`.github/workflows/release.yml:256-258`]().
2. **NPM Publish**: package를 npm registry에 publish합니다 [`.github/workflows/release.yml:251-255`]().
3. **GitHub Release**: version tag로 새 release를 만들고 모든 binary를 attach합니다 [`.github/workflows/release.yml:260-267`]().

## Post-Installation Logic

`postinstall.js` script(`npm install`을 통해 invoked)는 사용자 environment에 맞는 올바른 binary를 fetch하는 책임을 가집니다.

### Platform Detection
release workflow는 platform name을 target과 binary name(예: `agent-browser-linux-x64`, `agent-browser-darwin-arm64`)에 명시적으로 map합니다 [`.github/workflows/release.yml:75-117`](). 이를 통해 installation logic은 local system의 architecture와 OS에 따라 올바른 pre-built binary를 선택할 수 있습니다.

### Browser Installation
CLI의 `install` command는 "Chrome for Testing" download를 처리합니다 [`docs/src/app/installation/page.mdx:9`](). CI에서는 e2e test를 위한 environment를 provision하는 데 사용됩니다 [`.github/workflows/ci.yml:123-125`]().

### Debugging 및 Maintenance
platform-specific issue, 특히 Windows issue를 위해 repository에는 EC2 instance를 provision하고, code를 sync하며, remote build를 실행하는 debug script가 `scripts/windows-debug/`에 포함되어 있습니다 [`scripts/windows-debug/provision.sh`](), [`scripts/windows-debug/sync.sh:23-27`]().

Sources: [`.github/workflows/ci.yml:123-125`](), [`.github/workflows/release.yml:75-117`](), [`docs/src/app/installation/page.mdx:1-122`](), [`scripts/windows-debug/sync.sh:1-28`]()
