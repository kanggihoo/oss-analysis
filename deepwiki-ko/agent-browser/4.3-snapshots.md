## 목적과 범위

이 페이지는 element reference(ref)가 embedded된 웹 페이지의 accessibility tree representation을 생성하는 `agent-browser`의 snapshot system을 문서화합니다. snapshot은 AI 에이전트가 Chrome DevTools Protocol(CDP)의 `Accessibility` domain을 통해 page element를 발견하고 상호작용하는 기본 mechanism입니다.

command에서 ref가 사용되는 방식에 대한 정보는 [Element References (Refs)](/4.2)를 참조하세요. underlying communication layer는 [Communication Protocol](/3.5)을 참조하세요.

---

## 개요

snapshot은 웹 페이지에서 accessibility tree를 추출하고 element reference(`@e1`, `@e2` 등)로 보강합니다. modern web app에서 취약하거나 난독화될 수 있는 CSS selector와 달리, ref는 semantic role과 accessible name에서 생성되므로 AI가 element를 참조하는 결정적 방법을 제공합니다.

**핵심 특징:**
- **CDP-Native**: native daemon에서 `Accessibility` domain protocol을 직접 사용합니다 [cli/src/native/snapshot.rs:227-229]().
- **Deterministic**: 각 ref는 특정 `backend_node_id` 또는 semantic role/name pair를 가리킵니다 [cli/src/native/element.rs:9-16]().
- **AI-optimized**: output은 token usage를 최소화하도록 설계된 compact text format입니다 [docs/src/app/snapshots/page.mdx:33-42]().
- **Iframe Inlining**: iframe tree를 자동으로 traverse하고 main snapshot에 flatten합니다 [cli/src/native/snapshot.rs:221-222](), [docs/src/app/snapshots/page.mdx:81-96]().

**출처:** [cli/src/native/snapshot.rs:11-66](), [cli/src/native/element.rs:1-16](), [docs/src/app/snapshots/page.mdx:1-42]()

## Snapshot Generation Pipeline

다음 다이어그램은 `snapshot` command에서 `RefMap` 생성과 최종 text tree 생성까지의 흐름을 추적합니다.

### Snapshot Logic Flow
```mermaid
graph TB
    subgraph "Input_Space"
        ["CMD: agent-browser snapshot"]
        ["OPTS: SnapshotOptions"]
    end
    
    subgraph "Native_Snapshot_Logic"
        ["take_snapshot()"]
        ["CDP: Accessibility.enable"]
        ["IFRAME_MAP: iframe_sessions"]
        ["TREE_BUILDER: TreeNode"]
    end
    
    subgraph "Reference_Management"
        ["RefMap::add_with_frame()"]
        ["RefEntry_Struct"]
    end
    
    ["CMD: agent-browser snapshot"] --> ["take_snapshot()"]
    ["take_snapshot()"] --> ["CDP: Accessibility.enable"]
    ["CDP: Accessibility.enable"] --> ["IFRAME_MAP: iframe_sessions"]
    ["IFRAME_MAP: iframe_sessions"] --> ["TREE_BUILDER: TreeNode"]
    
    ["TREE_BUILDER: TreeNode"] --> ["RefMap::add_with_frame()"]
    ["RefMap::add_with_frame()"] --> ["RefEntry_Struct"]
    
    ["RefEntry_Struct"] --> ["OUTPUT: Compact_Text_Tree"]
```

**생성 단계:**
1. **Enable Domains**: daemon은 session에 대해 `DOM`과 `Accessibility` domain이 enabled 상태인지 보장합니다 [cli/src/native/snapshot.rs:224-229]().
2. **Fetch AXTree**: selector가 제공되면 `DOM.describeNode`를 통해 scope를 지정하거나 [cli/src/native/snapshot.rs:259-264](), `Accessibility.getFullAXTree`를 통해 `AXNode` object의 tree를 가져옵니다 [cli/src/native/snapshot.rs:7-8]().
3. **Iframe Traversal**: `Iframe` node가 발견되면, 시스템은 `iframe_sessions`를 사용해 해당 iframe 전용 CDP session을 resolve합니다 [cli/src/native/snapshot.rs:222]().
4. **Ref Assignment**: `INTERACTIVE_ROLES` 또는 `CONTENT_ROLES`와 일치하는 element에는 unique `@eN` reference가 할당됩니다 [cli/src/native/snapshot.rs:11-43]().
5. **Caching**: 각 ref의 `backend_node_id`, `role`, `name`, `frame_id` context가 `RefMap`에 저장됩니다 [cli/src/native/element.rs:42-62]().

**출처:** [cli/src/native/snapshot.rs:216-270](), [cli/src/native/element.rs:42-62](), [cli/src/native/snapshot.rs:11-43]()

## Element Classification

snapshot system은 ARIA role을 기준으로 visibility와 ref assignment를 결정하기 위해 element를 분류합니다.

### Interactive Roles
이 element들은 항상 ref를 받으며 `--interactive` mode에서 보존됩니다 [cli/src/native/snapshot.rs:11-30]().
- `button`, `link`, `textbox`, `checkbox`, `radio`
- `combobox`, `listbox`, `menuitem`, `searchbox`
- `slider`, `spinbutton`, `switch`, `tab`, `treeitem`, `Iframe`

### Content & Structural Roles
- **Content Roles**: `heading`, `cell`, `gridcell`, `article`, `main`, `navigation`. 의미 있는 text를 포함하면 ref를 받습니다 [cli/src/native/snapshot.rs:32-43]().
- **Structural Roles**: `generic`, `group`, `list`, `table`, `row`, `WebArea`, `RootWebArea`. tree hierarchy에 필요하지 않은 한 일반적으로 `--compact` mode에서 생략됩니다 [cli/src/native/snapshot.rs:45-66]().

**출처:** [cli/src/native/snapshot.rs:11-66]()

## Filtering and Transformation

### Interactive-Only Mode (`-i`)
`options.interactive`가 true이면 tree는 `INTERACTIVE_ROLES`에 정의된 element만 포함하도록 pruning됩니다 [cli/src/native/snapshot.rs:80](), [docs/src/app/snapshots/page.mdx:11]().

### Compact Mode (`-c`)
accessible name이나 할당된 reference가 없는 빈 structural element(예: `generic`, `group`)를 제거하여 AI를 위해 tree를 flatten합니다 [cli/src/native/snapshot.rs:81](), [docs/src/app/snapshots/page.mdx:25]().

### Iframe Inlining
시스템은 browser의 multi-process frame architecture를 단일 tree로 flatten합니다.
- **Cross-Origin Handling**: out-of-process iframe의 경우, 시스템은 `iframe_sessions`를 사용해 해당 `session_id`로 child frame의 AXTree를 query합니다 [cli/src/native/snapshot.rs:222]().
- **Context Preservation**: `RefMap`의 각 ref는 자신이 originate된 `frame_id`를 저장하므로, `click @e5` 같은 command가 올바른 CDP session을 자동으로 target할 수 있습니다 [cli/src/native/element.rs:15-16](), [cli/src/native/element.rs:42-62]().

**출처:** [cli/src/native/snapshot.rs:77-84](), [cli/src/native/element.rs:9-16](), [docs/src/app/snapshots/page.mdx:81-96]()

## RefMap System

`RefMap`은 AI가 제공한 ref를 browser action으로 다시 변환하기 위한 source of truth입니다.

### Ref Data Structure
| Field | Type | Description |
|-------|------|-------------|
| `backend_node_id` | `Option<i64>` | stable CDP node identifier [cli/src/native/element.rs:10](). |
| `role` | `String` | ARIA role(예: "button") [cli/src/native/element.rs:11](). |
| `name` | `String` | accessible name(예: "Submit") [cli/src/native/element.rs:12](). |
| `frame_id` | `Option<String>` | element를 포함하는 frame의 ID [cli/src/native/element.rs:15](). |

### Ref Resolution Space
이 다이어그램은 CLI의 ref string이 internal function과 CDP type을 사용해 screen의 physical location으로 resolve되는 방식을 보여줍니다.

```mermaid
graph LR
    subgraph "Natural_Language_Space"
        ["REF_ID: @e1"]
    end
    
    subgraph "Code_Entity_Space"
        ["RefMap::get()"]
        ["RefEntry"]
        ["resolve_element_center()"]
        ["resolve_element_object_id()"]
    end
    
    subgraph "Browser_CDP_Space"
        ["CDP: DOM.getBoxModel"]
        ["CDP: DOM.resolveNode"]
        ["FUNC: find_node_id_by_role_name"]
    end
    
    ["REF_ID: @e1"] --> ["RefMap::get()"]
    ["RefMap::get()"] --> ["RefEntry"]
    ["RefEntry"] --> ["resolve_element_center()"]
    ["RefEntry"] --> ["resolve_element_object_id()"]
    
    ["resolve_element_center()"] --> ["CDP: DOM.getBoxModel"]
    ["resolve_element_object_id()"] --> ["CDP: DOM.resolveNode"]
    
    ["CDP: DOM.getBoxModel"] -- "Fallback" --> ["FUNC: find_node_id_by_role_name"]
    ["FUNC: find_node_id_by_role_name"] --> ["CDP: DOM.getBoxModel"]
```

**Resolution Logic:**
1. **Fast Path**: cached `backend_node_id`와 `DOM.getBoxModel` 사용을 시도합니다 [cli/src/native/element.rs:165-181]().
2. **Fallback Path**: `backend_node_id`가 stale한 경우(예: DOM mutation), 시스템은 저장된 `role`과 `name`으로 `find_node_id_by_role_name`을 사용해 semantic lookup을 수행합니다 [cli/src/native/element.rs:186-195]().

**출처:** [cli/src/native/element.rs:9-21](), [cli/src/native/element.rs:149-209](), [cli/src/native/interaction.rs:37-44]()

## Annotated Screenshots

visual context를 위해 `agent-browser`는 `screenshot --annotate` flag를 사용해 ref를 screenshot 위에 직접 overlay할 수 있습니다 [cli/src/native/screenshot.rs:61](), [docs/src/app/snapshots/page.mdx:66]().

1. **Collection**: 시스템은 `collect_annotations`를 통해 모든 interactive element와 해당 bounding box를 가져옵니다 [cli/src/native/screenshot.rs:120-123](), [cli/src/native/screenshot.rs:231-235]().
2. **Injection**: `ANNOTATION_OVERLAY_ID`("__agent_browser_annotations__")가 정의한 temporary overlay가 `inject_annotation_overlay`를 사용해 page DOM에 inject됩니다 [cli/src/native/screenshot.rs:11](), [cli/src/native/screenshot.rs:126-131]().
3. **Capture**: overlay가 보이는 동안 `Page.captureScreenshot`이 호출됩니다 [cli/src/native/screenshot.rs:224-226]().
4. **Cleanup**: capture 직후 `remove_annotation_overlay`를 사용해 overlay가 제거됩니다 [cli/src/native/screenshot.rs:136-138]().

**출처:** [cli/src/native/screenshot.rs:100-169](), [cli/src/native/screenshot.rs:171-229](), [docs/src/app/snapshots/page.mdx:64-80]()
